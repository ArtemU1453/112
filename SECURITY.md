# Политика безопасности — ЕАСУР

ЕАСУР — государственная система реагирования экстренных служб. Безопасность имеет высший
приоритет (Security by Design; Project Constitution, ARTICLE 10; Security Standards).

## Сообщение об уязвимости
- **Не** создавайте публичный issue для уязвимостей.
- Сообщайте об уязвимостях по внутренним каналам ответственной группы безопасности организации
  (защищённый канал, определяемый администрацией проекта).
- Укажите: описание, шаги воспроизведения, потенциальное влияние, версии/компоненты.

## Обработка
- Подтверждение получения — в кратчайший срок.
- Оценка серьёзности и приоритизация; для критических/высоких уязвимостей — ускоренный процесс
  (Emergency Release, Release Governance Standard).
- Исправление сопровождается регистрацией в Backlog и, при необходимости, записью в Risk Register.

## Поддерживаемые версии
Поддерживается актуальная линия релизов согласно каталогу версий
(`config/toolchain/versions.yaml`) и `deprecated-versions.yaml`. Компоненты в статусе
End of Life не поддерживаются.

## Требования безопасности (сводка)
- Секреты не хранятся в репозитории (Repository/Environment Governance Standards).
- Обязательные проверки цепочки поставки: secret/dependency/container scanning, license, SBOM
  (Security Standards; CI-проверки).
- Аутентификация OIDC, авторизация RBAC, минимальные привилегии, аудит действий.

## Раскрытие
Ответственное раскрытие: детали уязвимости публикуются только после выпуска исправления и по
согласованию с ответственной группой безопасности.
