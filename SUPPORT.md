# Поддержка — ЕАСУР

## Документация
- Индекс документации: `docs/README.md`.
- Быстрый старт: `docs/developer-experience/quick-start.md`.
- Установка: `docs/developer-experience/installation.md`.
- Локальная разработка/отладка: `docs/developer-experience/local-development.md`, `debug.md`.
- Эксплуатация: `docs/operations.md`.

## Вопросы и обсуждения
- Технические вопросы — через GitHub Discussions (шаблоны в `.github/DISCUSSION_TEMPLATE/`).
- Дефекты — через Issues (шаблон Bug Report).
- Предложения функциональности — через Issues (шаблон Feature Request).
- Изменения архитектуры — через RFC (`docs/rfc/`), не через обычный issue.

## Что не относится к поддержке здесь
- Уязвимости безопасности — только через `SECURITY.md` (не публичные issue).
- Эксплуатационные инциденты промышленного контура — по внутренним каналам эксплуатации.

## Правила обращения
Соблюдайте `CODE_OF_CONDUCT.md`. Для воспроизводимых проблем прикладывайте версию/компонент,
шаги воспроизведения и логи без чувствительных данных (Logging Standards).
