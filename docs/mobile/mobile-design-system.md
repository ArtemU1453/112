# Mobile Design System — ЕАСУР

- **Status:** APPROVED · Stage 7 · Подчиняется Mobile Accessibility Standard, UX. Единая для всех
  мобильных приложений экосистемы.

## Назначение
Единый Mobile Design System — обязательная система компонентов, цветов и правил для **всех**
мобильных приложений (единый дизайн — проверка этапа). Токены — `libraries/mobile-design-tokens/`.

## Компоненты
Единый набор: кнопки (крупные), поля ввода, списки/карточки инцидентов/подразделений, статусные
чипы, карта, панель статусов, вложения/медиа, формы, диалоги подтверждения. Компоненты —
переиспользуемые, единообразные во всех приложениях.

## Цвета состояний (единые)
Соответствуют домену (приоритеты/статусы). Определены токенами:
- Приоритеты: CRITICAL (красный), HIGH (оранжевый), MEDIUM (жёлтый), LOW (зелёный).
- Статусы подразделения: AVAILABLE (зелёный), EN_ROUTE (синий), ON_SCENE (фиолетовый),
  OUT_OF_SERVICE (серый).
Единые семантические цвета обеспечивают одинаковую интерпретацию во всех приложениях.

## Темы
- **Дневной режим** (светлая тема), **ночной режим** (тёмная тема), **режим высокой контрастности**.
- Переключение — вручную и по системной настройке/времени; ночной режим — для работы в темноте.

## Доступность (7 UX / Mobile Accessibility Standard)
- **Работа в перчатках:** увеличенные зоны нажатия (минимальный размер сенсорной цели ≥ 48dp).
- **Крупные элементы управления** и шрифты; масштабируемый текст.
- Высокий контраст; поддержка программ чтения с экрана; понятные состояния фокуса.
- Минимизация точности ввода (крупные списки/кнопки вместо мелких элементов).

## Токены (единый источник)
`libraries/mobile-design-tokens/tokens.json` — цвета (light/dark/high-contrast), типографика,
отступы, размеры сенсорных целей, радиусы. Приложения используют токены, не хардкодят значения.

## Соответствие
Mobile Accessibility Standard, UX-требования этапа; единый дизайн обязателен во всех приложениях
(Definition of Done этапа).
