# Mobile Architecture — ЕАСУР

- **Status:** APPROVED · Stage 7 · Подчиняется Architecture.md, Core Platform Architecture, EAGF,
  ADR-016/017/018. Единая мобильная экосистема полевых операций и командования.

## 1. Назначение
Единая мобильная экосистема для подразделений и руководящего состава: полевые приложения,
приложение начальника караула, рабочее место руководителя ликвидации ЧС. **Все приложения
используют единую платформу, единый SDK, единые API и политики безопасности** (требование этапа).

## 2. Приложения (роли)
| Приложение | Каталог | Роль (7.x) |
|-----------|---------|-----------|
| Field Responder | `apps/mobile-crew` | Подразделения (7.1) |
| Commander | `apps/mobile-commander` | Начальник караула (7.2) |
| Incident Command | `apps/mobile-command` | Руководитель ликвидации ЧС / штаб (7.3) |

Все три — на **едином кросс-платформенном стеке** (ADR-016) с общим Mobile SDK и дизайн-системой.

## 3. Слои мобильного приложения
```
UI (Mobile Design System) → App logic (роль-специфичные экраны)
  → Mobile SDK (security, sync, messaging, media, telemetry, forms, audit, config)
  → Local encrypted store (offline-first) ↕ Sync engine (outbox/delta/versions)
  → Platform APIs (общие контракты, через Gateway) ↔ Backend services (Stage 0/4/5/6)
```
Бизнес-логика домена **не дублируется** на клиенте — используется платформа (запрет этапа);
клиент содержит представление, offline-кэш и координацию.

## 4. Переиспользование платформы
- **API:** мобильные приложения используют **общие** контракты (`contracts/`), а не app-специфичные
  (запрет отдельных API под одно приложение без ADR). Через Gateway (JWT, ADR-007).
- **SDK:** общий Mobile SDK (security/logging/config/audit/errors/monitoring/tracing — аналог
  Platform SDK для мобильного стека; Mobile SDK Guide).
- **Идентичность:** OIDC/JWT (Keycloak, ADR-007); RBAC-роли (`Roles`); service accounts устройств.

## 5. Offline-First (7.4/7.5)
Полная работа без сети; локальное шифрованное хранилище; очередь синхронизации (outbox);
двусторонняя дельта-синхронизация с версионированием, проверкой целостности, повторной передачей,
разрешением конфликтов (ADR-017; Synchronization Architecture, Offline Synchronization Standard).

## 6. Возможности (7.1–7.10) и стандарты
Карточка выезда/маршрут/навигация/статусы/медиа (7.1); состав/команды/усиление (7.2); управление
силами/задачи/журналы/карта/безопасность (7.3); offline (7.4/7.5); операционные сообщения (7.6);
цифровые формы (7.7); медиа (7.8); полевая телеметрия (7.9); MDM (7.10). Каждая — стандартом и/или
общим контрактом (см. docs/README, раздел Mobile).

## 7. Безопасность
Шифрование локального хранилища (SQLCipher/эквивалент), защищённое хранение ключей (Keystore),
проверка целостности приложения, обнаружение модификации, авто-завершение сессии, журнал
безопасности; интеграция с MDM (ADR-018). Секреты не хранятся в коде приложения (запрет этапа).
Детали — Mobile Security / Device Security Standards.

## 8. Наблюдаемость и аудит
Единые логи/метрики/трассы через Mobile SDK (Operational Telemetry Standard); мобильный аудит
действий (Mobile Audit Standard) синхронизируется с платформенным аудитом.
