# ADR-001 — Язык программирования backend

- **Статус:** Accepted
- **Дата:** Project Start
- **Контекст-ссылки:** Vision (Ц1, Ц6), Architecture Principles

## Контекст

Требуется промышленный государственный backend с длительным жизненным циклом, высокими
требованиями к надёжности, безопасности, доступности 24/7 и наличию квалифицированных
кадров сопровождения. Экосистема должна включать зрелые решения для веб-сервисов, доступа
к данным, безопасности, интеграции с Kafka и наблюдаемости. Отдельный сервис обработки
речи/текста (AI) имеет иные требования (ML-экосистема).

## Рассматриваемые варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Java 21 (LTS) + Spring Boot** | Зрелая enterprise-экосистема, LTS, сильная типизация, виртуальные потоки, широкий рынок кадров, богатые библиотеки безопасности/данных/Kafka | Многословность, потребление памяти JVM |
| Kotlin + Spring | Лаконичнее Java, совместимость с JVM | Меньше кадров в госсекторе; выигрыш не критичен |
| Go | Малое потребление ресурсов, простые бинарники | Беднее экосистема enterprise-безопасности/ORM; больше ручного кода |
| Node.js/TypeScript | Единый язык с frontend | Слабее для нагруженного доменного backend, типобезопасность слабее |
| Python (для всего backend) | Простота, ML | Не оптимален для нагруженного транзакционного ядра |

## Принятое решение

Backend прикладных доменных сервисов — **Java 21 (LTS) + Spring Boot 3.3**. Отдельный
**ai-service** реализуется на **Python (FastAPI)** ввиду ML-экосистемы (обработка речи/текста)
— это осознанный полиглот-компромисс, изолированный границей сервиса и контрактом (Kafka/REST).

## Обоснование

- **Проблема:** нужен надёжный долгоживущий backend с сильной экосистемой безопасности,
  данных и интеграции.
- **Почему Java 21:** LTS-поддержка на годы вперёд; зрелые Spring Security/Data/Kafka;
  строгая типизация снижает класс ошибок; виртуальные потоки (Project Loom) улучшают
  обработку конкурентных вызовов; широкий рынок специалистов в госсекторе.
- **Преимущества:** предсказуемость сопровождения, безопасность, наблюдаемость, найм.
- **Недостатки/компромиссы:** многословность и память JVM; принято ради надёжности и
  экосистемы. Полиглот (Python для AI) добавляет вторую цепочку сборки — компенсируется
  чёткой изоляцией сервиса.
- **Влияние на развитие:** единый стек Java для доменных сервисов упрощает стандарты кода,
  тестирования и эксплуатации.
- **Условия пересмотра:** смена LTS-линии Java; появление требований, где JVM неприемлема
  по ресурсам, — через новый RFC/ADR.

## Последствия

- Единые Coding Standards для Java (см. `docs/standards/coding-standards.md`).
- ai-service живёт по собственным стандартам Python, но соблюдает общие контракты и Quality Gates.
