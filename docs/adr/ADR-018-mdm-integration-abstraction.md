# ADR-018 — Абстракция интеграции с MDM

- **Статус:** Accepted
- **Дата:** 2026-07-21
- **Связи:** Mobile Security Standard, Device Security Standard, TAP, Capability `CAP-MDM-001`

## Контекст

Stage 7 требует слоя интеграции с системами управления мобильными устройствами (MDM/EMM):
регистрация устройств, удалённая блокировка, удалённое стирание корпоративных данных, контроль
соответствия политике безопасности, управление сертификатами. Не должно быть привязки к
конкретному поставщику MDM; система суверенна (возможен ведомственный/self-hosted MDM).

## Рассматриваемые варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Порт `MdmProvider` + адаптеры (Android Enterprise / ведомственный MDM)** | Независимость от поставщика (TAP), единый контракт, суверенность | Требует адаптеров под конкретные MDM |
| Жёсткая интеграция с одним MDM | Быстрее | Vendor lock; неприемлемо |
| Собственный MDM с нуля | Полный контроль | Дорого, дублирует зрелые решения |

## Принятое решение

**Абстракция MDM через порт `MdmProvider`** (`CAP-MDM-001`) с адаптерами к конкретным системам
(Android Enterprise/AMAPI, ведомственный/self-hosted MDM). Единый контракт интеграции
(`contracts/internal/mdm-integration-api.yaml`): регистрация устройства, политики соответствия,
удалённая блокировка/стирание корпоративных данных, управление сертификатами. Конкретный MDM не
фиксируется в прикладном коде (TAP); распространение приложений — через MDM/внутренний канал (не
публичные сторы обязательно).

## Обоснование

- **Проблема:** управление устройствами без привязки к поставщику и с суверенностью.
- **Почему абстракция:** соблюдение TAP; заменяемость MDM без изменения приложений; единые политики.
- **Почему не жёсткая интеграция/свой MDM:** vendor lock либо чрезмерная стоимость.
- **Компромиссы:** адаптеры под каждый MDM; смягчается единым контрактом и Device Security Standard.
- **Условия пересмотра:** смена MDM — адаптером и Technology Mapping; несовместимость — RFC/ADR.

## Последствия

- Контракт MDM-интеграции; порт — в мобильной платформе/адаптерах.
- `CAP-MDM-001` в Technology Mapping; управление сертификатами согласовано с секрет-менеджментом
  (ADR-011). Секреты не хранятся в коде приложения (запрет этапа).
