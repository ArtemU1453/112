# ADR-021 — Абстракция аналитического ИИ и объяснимость

- **Статус:** Accepted
- **Дата:** 2026-07-21
- **Связи:** AI Analytics Framework, AI Explainability / Decision Intelligence / Forecasting
  Standards, ADR-012 нет — AI Provider Abstraction Standard (Stage 4), TAP, Capability
  `CAP-AIANALYTICS-001`

## Контекст

Stage 8 создаёт универсальный аналитический слой ИИ: классификация, кластеризация, поиск
закономерностей, выявление отклонений, генерация сводок, **объяснимость**. Явные требования:
модели подключаются через **адаптеры**; ИИ — **только инструмент анализа/рекомендаций**, не
основание для автоматических управленческих решений; все выводы сопровождаются объяснением и
указанием использованных данных; запрещены модели без документированной методики.

## Рассматриваемые варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Порт `AnalyticsModel` + адаптеры (self-hosted модели/статистика) с обязательной методикой и объяснимостью** | Независимость от модели (TAP), суверенность, объяснимость, документированная методика | Требует дисциплины методик/адаптеров |
| Жёсткая интеграция с одной ML-платформой | Быстрее | Vendor lock; недостаточно объяснимости/суверенности |
| Только «чёрный ящик» без объяснений | Простота | Нарушает требования объяснимости/методики; неприемлемо |

## Принятое решение

**Абстракция аналитического ИИ через порт `AnalyticsModel`** (`CAP-AIANALYTICS-001`),
расширяющий подход AI Provider Abstraction Standard (Stage 4). Модели/методы (классификация,
кластеризация, выявление закономерностей/отклонений, прогноз, суммаризация) подключаются
адаптерами; каждая — с **документированной методикой** (Statistical Methodology Standard) и
**объяснимостью** (AI Explainability Standard: объяснение + перечень использованных данных +
показатель неопределённости). Модели self-hosted (суверенность, AGSDS). **ИИ не принимает
управленческие решения автоматически** — только предоставляет анализ/рекомендации человеку.

## Обоснование

- **Проблема:** аналитический ИИ без vendor lock, с объяснимостью и методикой, суверенно.
- **Почему абстракция + обязательная методика/объяснимость:** соответствие запретам этапа (нет
  «чёрных ящиков» без методики; нет автономных решений); заменяемость моделей (TAP).
- **Почему не жёсткая интеграция/чёрный ящик:** vendor lock либо нарушение объяснимости.
- **Компромиссы:** обязательность методики/объяснения для каждой модели; смягчается стандартами и
  каталогом метрик/lineage.
- **Условия пересмотра:** смена модели/метода — адаптером + методикой + Technology Mapping; иное — RFC/ADR.

## Последствия

- Существующий ai-service — один из адаптеров; аналитические модели — в `analytics-service`.
- Каждый результат ИИ содержит объяснение, использованные данные и неопределённость (контракты
  `ai-analytics`, `decision-intelligence`, `forecasting`).
- `CAP-AIANALYTICS-001` в Technology Mapping; версии — каталог (TVMS).
