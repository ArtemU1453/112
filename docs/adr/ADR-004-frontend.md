# ADR-004 — Frontend

- **Статус:** Accepted
- **Дата:** Project Start
- **Связи:** ADR-007 (OIDC), Architecture.md (клиентский уровень)

## Контекст

АРМ диспетчера — насыщенное одностраничное приложение реального времени: карта обстановки,
таблицы происшествий/подразделений, детальные карточки, формы, обновления по WebSocket,
аутентификация через OIDC. Нужны предсказуемое управление состоянием, картография и зрелая
UI-библиотека, а также совместимость с корпоративной аутентификацией.

## Рассматриваемые варианты

| Аспект | Варианты | Выбор |
|--------|----------|-------|
| Каркас | React, Angular, Vue | **React 19 + TypeScript** |
| Управление состоянием | Redux Toolkit, MobX, Zustand | **Redux Toolkit** |
| UI-библиотека | MUI, Ant Design, custom | **MUI** |
| Картография | OpenLayers, Leaflet, MapLibre | **OpenLayers** |
| Реальное время | STOMP/SockJS, native WS, SSE | **STOMP over SockJS** |
| Сборка | Vite, CRA, Webpack | **Vite** |
| Auth | Keycloak OIDC (PKCE) | **OIDC + PKCE S256** |

## Принятое решение

**React 19 + TypeScript**, **Redux Toolkit** (срезы incidents/units/calls), **MUI** (в т.ч.
DataGrid), **OpenLayers** (карта), **STOMP over SockJS** (реальное время), сборка **Vite**,
аутентификация — **Keycloak OIDC с PKCE (S256)**. Прод-образ — многостадийная сборка
(Node → nginx) с проксированием `/api` и `/ws`.

## Обоснование

- **Проблема:** нужен зрелый насыщенный SPA реального времени с картой и корпоративной auth.
- **Почему React + TS:** крупнейшая экосистема и рынок кадров; строгая типизация контрактов;
  предсказуемость сопровождения.
- **Почему Redux Toolkit:** явная, отлаживаемая модель состояния для потоков реального времени
  (обновления из WebSocket) против неявной реактивности; стандартизованные паттерны.
- **Почему MUI:** готовый набор компонентов (включая DataGrid) ускоряет АРМ без собственной
  дизайн-системы; тёмная тема для операторского зала.
- **Почему OpenLayers:** мощные возможности для оперативной карты (слои, проекции, векторные
  стили) сверх простого Leaflet.
- **Почему STOMP/SockJS:** совместимость с брокерным WebSocket сервера realtime; деградация
  через SockJS при ограничениях сети.
- **Недостатки/компромиссы:** размер бандла React/MUI/OpenLayers — приемлемо для рабочего места
  в контролируемой сети; Redux многословнее — компенсируется предсказуемостью.
- **Условия пересмотра:** появление мобильных клиентов не отменяет данное решение для АРМ;
  смена — через RFC/ADR.

## Последствия

- Стандарты TypeScript — см. `docs/standards/coding-standards.md`.
- Клиент обращается только через Gateway (`/api`, `/ws`).
