# ADR-026 — Национальный цифровой двойник и платформа моделирования

- **Статус:** Accepted
- **Дата:** 2026-07-21
- **Связи:** Digital Twin Architecture, Simulation Architecture, Digital Twin Governance Standard,
  Simulation Security Standard, Synthetic Data Standard, ADR-002, ADR-021, ADR-025, EAGF,
  Capability `CAP-DTWIN-001`, `CAP-SIM-001`

## Контекст

Для стратегического планирования, обучения и безопасной проверки изменений требуется **цифровой
двойник** национальной системы реагирования и платформа моделирования. Ключевое требование —
**полная изоляция от промышленной эксплуатации**: вмешательство в рабочую систему на этом этапе
запрещено, все моделирование ведётся в отдельной среде на синтетических/обезличенных данных. Нужно
зафиксировать архитектуру и границу изоляции, не нарушая Architecture Freeze.

## Рассматриваемые варианты

| Аспект | Варианты | Выбор |
|--------|----------|-------|
| Развёртывание | Общая среда с prod vs **полностью изолированная среда (отдельный контур)** | **Изолированная среда** |
| Данные | Копия боевых данных vs **синтетические/обезличенные (Synthetic Data Standard)** | **Синтетические/обезличенные** |
| Связь с prod | Двунаправленная vs **однонаправленный, только офлайн-импорт справочной модели** | **Однонаправленный импорт** |
| Технология | Фиксированный продукт vs **self-hosted, абстракция возможности (TAP)** | **Self-hosted, capability-first** |
| Перенос результатов | Автоматический vs **только через утверждённую процедуру (RFC→ARB→ADR)** | **Только через утверждение** |

## Принятое решение

**Отдельная изолированная платформа цифрового двойника и моделирования** (`CAP-DTWIN-001`,
`CAP-SIM-001`):
- Развёртывается в **изолированном контуре**, физически/логически отделённом от промышленного;
  сетевого пути влияния на прод нет (Simulation Security Standard).
- Работает на **синтетических/обезличенных** данных (Synthetic Data Standard); использование
  синтетических данных как реальных запрещено; смешение учебных и реальных инцидентов запрещено.
- Импорт из прод — **только однонаправленный офлайн-снимок** справочной модели (подразделения/
  территории/инфраструктура) для построения двойника; обратная запись в прод отсутствует.
- **Перенос любых результатов** моделирования/экспериментов в прод — только через утверждённую
  процедуру (RFC → Architecture Review Board → ADR, ADR-025); автоматический перенос запрещён.
- Технологии — **self-hosted, абстрагированы возможностью** (TAP); суверенность сохраняется (AGSDS).

## Обоснование

- **Проблема:** нужна доказуемая безопасность экспериментов над национальной системой без риска для
  эксплуатации.
- **Почему изоляция + синтетика:** исключает влияние на прод и утечки; воспроизводимость на
  контролируемых данных.
- **Почему capability-first:** без вендор-лока; движок двойника/симуляции заменяем через ADR.
- **Компромиссы:** отдельная инфраструктура и синхронизация справочной модели; оправдано критичностью.
- **Условия пересмотра:** изменение границы изоляции/модели данных — через RFC/ADR.

## Последствия

- Появляется отдельный контур (двойник/симуляция/обучение/стратегическое планирование) вне прод.
- Экспериментальные модели ИИ проверяются здесь (ADR-027) и не попадают в прод без утверждения.
- `CAP-DTWIN-001`/`CAP-SIM-001` вносятся в Technology Mapping; версии — в каталоге (раздел simulation).
