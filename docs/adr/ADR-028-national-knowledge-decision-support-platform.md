# ADR-028 — Национальная платформа знаний и поддержки решений

- **Статус:** Accepted
- **Дата:** 2026-07-21
- **Связи:** Knowledge Platform Architecture, Regulatory Document Standard, Rules Catalog Specification,
  Semantic Search Governance Standard, AI Assistance Policy, AI Explainability Standard, ADR-021,
  ADR-025, EAGF, Capability `CAP-KNOW-001`, `CAP-SEARCH-001`

## Контекст

Диспетчерам и руководителям нужен **единый источник** нормативной информации, стандартных операционных
процедур (SOP), справочников и правил поддержки решений — версионируемый, объяснимый и контролируемо
используемый. Разрозненные документы создают риск применения неутверждённых материалов и правил без
основания. Требуется зафиксировать платформу знаний и её инварианты, не меняя бизнес-логику ранее
реализованных модулей без RFC→ARB→ADR. Возможность универсального поиска (`CAP-SEARCH-001`) оставалась
Proposed с Stage 4 — назрело её утверждение для платформы знаний.

## Рассматриваемые варианты

| Аспект | Варианты | Выбор |
|--------|----------|-------|
| Источник нормативной информации | Разрозненные хранилища vs **единый реестр (SSOT знаний)** | **Единый реестр** |
| Использование правил | Свободное vs **только с документированным основанием (нормативная ссылка)** | **С основанием** |
| Поиск | Только полнотекст vs **полнотекст + атрибуты + семантика (адаптер ИИ)** | **Комбинированный** |
| Поисковый движок | Фиксированный продукт vs **self-hosted, абстракция (`CAP-SEARCH-001` → Approved)** | **Self-hosted, абстракция** |
| Поддержка решений | ИИ решает vs **рекомендации из утверждённых SOP, объяснимые, подтверждаемые человеком** | **Совещательные, объяснимые** |

## Принятое решение

**Единая национальная платформа знаний и поддержки решений** (`CAP-KNOW-001`):
- **SSOT нормативной информации:** реестр документов/SOP/справочников/правил; используются **только
  утверждённые** версии; история изменений неизменяема (не удаляется).
- **Объяснимость правил:** каждое правило имеет **документированное основание** (ссылку на норматив);
  применение правил без основания запрещено.
- **Поиск:** полнотекстовый + по атрибутам + **семантический через абстрактный адаптер ИИ**
  (`CAP-AI-001`); поисковый движок — **self-hosted, абстрагирован**; `CAP-SEARCH-001` переводится в
  **Approved** (движок не фиксируется, выбирается реализацией).
- **Поддержка решений:** рекомендации формируются из **утверждённых SOP**, сопровождаются объяснением
  и (при использовании ИИ) уровнем уверенности; **ИИ остаётся совещательным** (AI Assistance Policy) —
  окончательное решение подтверждает пользователь. Автоматически менять рабочие процессы запрещено.
- **Распространение изменений:** контролируемо (уведомление, обязательное ознакомление, журнал),
  только через утверждённый процесс согласования/публикации.

## Обоснование

- **Проблема:** единообразие, актуальность, объяснимость и контролируемость знаний в критической системе.
- **Почему SSOT + основания + объяснимость:** исключает применение неутверждённых материалов и
  необоснованных правил; полная трассируемость.
- **Почему абстрактный поиск:** без вендор-лока (TAP); суверенность (AGSDS).
- **Компромиссы:** дисциплина согласования/версионирования; оправдано критичностью.
- **Условия пересмотра:** выбор конкретного поискового движка/семантической модели — через RFC/ADR с
  обновлением Technology Mapping.

## Последствия

- Появляется платформа знаний (реестр/SOP/правила/справочники/поиск/распространение/аналитика).
- Бизнес-логика оперативных модулей **не меняется**; интеграция — как справочный слой (только чтение
  утверждённых знаний); изменения процессов — через RFC→ARB→ADR.
- `CAP-KNOW-001` добавлена; `CAP-SEARCH-001` переведена в Approved в Technology Mapping (снимает часть
  TD-2026-001). ИИ — через существующую абстракцию (ADR-021), объяснимость (AI Explainability).
