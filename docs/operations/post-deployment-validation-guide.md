# Post Deployment Validation Guide — ЕАСУР

- **Status:** APPROVED · Stage 10 · Реализует подэтапы 10.1, 10.7. Проверки после каждого выката и
  рубежа go-live (Deployment Program, Go-Live Playbook).

## 1. Назначение
Подтвердить корректность и работоспособность системы сразу после развёртывания, до включения/расширения
трафика. Невыполнение блокирующих проверок → откат (Go-Live Playbook).

## 2. Технические проверки
- [ ] Все поды Healthy; readiness/liveness проходят; версии образов соответствуют релизу (TVMS).
- [ ] Миграции применены без ошибок; схема соответствует ожидаемой.
- [ ] Зависимости доступны: БД/PostGIS, Kafka, Redis, Keycloak, объектное хранилище, интеграции.
- [ ] Prometheus targets активны; логи/трассы поступают; алерты не в состоянии сбоя.

## 3. Функциональные smoke-проверки (критический путь)
- [ ] Приём вызова (telephony) → регистрация инцидента (incident) — сквозной сценарий.
- [ ] Классификация и рекомендации ИИ доступны и **совещательны** (решение — за диспетчером).
- [ ] Диспетчеризация подразделения (dispatch) и обновление статусов.
- [ ] Геокодирование/карта (gis) и телеметрия отображаются.
- [ ] Уведомления и realtime-обновления доставляются; аудит фиксирует действия.

## 4. Нефункциональные проверки
- [ ] Латентность/ошибки критического пути в пределах SLO (сверка с сертификацией производительности).
- [ ] Безопасность: mTLS/подпись интеграций активны; авторизация действует; секреты из Vault.

## 5. Данные и восстановление
- [ ] Резервная копия после выката снимается; выборочная проверка целостности.
- [ ] Согласованность данных между сервисами (событийные потоки без потерь).

## 6. Итог
Все блокирующие проверки пройдены → допуск к включению/расширению трафика на рубеже; результат —
в протокол рубежа (Go-Live Playbook) и Post Deployment отчётность.
