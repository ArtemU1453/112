# Production Runbook — ЕАСУР

- **Status:** APPROVED · Stage 10 · Реализует подэтап 10.8. Дополняет платформенные
  [Runbooks](../platform/runbooks/README.md) прикладными процедурами эксплуатации сервисов ЕАСУР.

## 1. Область
Пошаговые процедуры эксплуатации и восстановления прикладного контура (9 сервисов + ai-service +
frontend). Платформенные процедуры (кластер, хранилище, сеть) — в platform/runbooks.

## 2. Диагностика (общая последовательность)
1. Дашборды SLA/SLO и алерты (Grafana) → локализовать затронутый сервис/путь.
2. Health/readiness подов; логи (Loki) и трассы (Tempo/OTel) по `correlationId`.
3. Состояние зависимостей: БД (PostgreSQL/PostGIS), брокер (Kafka), кэш (Redis), идентификация
   (Keycloak), интеграции (платформа/mTLS).

## 3. Типовые процедуры
| Ситуация | Действия | Критерий восстановления |
|----------|----------|-------------------------|
| Деградация сервиса | Проверить ресурсы/HPA, перезапуск пода, откат релиза | SLA восстановлен |
| Отставание Kafka-consumer | Проверить лаг/партиции, масштабирование consumer | Лаг в норме |
| Ошибки БД/миграций | Проверить миграции (Liquibase), соединения, откат релиза | Запросы успешны |
| Сбой интеграции | Проверить сертификаты/mTLS/реестр, повторы/DLQ | Обмен восстановлен |
| Недоступность идентификации | Проверить Keycloak/realm, токены | Аутентификация работает |

## 4. Откат релиза
`helm rollback` на предыдущую версию; проверка миграций на обратимость; post-deployment validation;
при повреждении данных — восстановление (Backup Policy/DR).

## 5. Эскалация
По [Incident Escalation Standard](../standards/Incident-Escalation-Standard.md); SEV-1/2 — координатор
инцидента и, при необходимости, DR-координация.

## 6. После инцидента
Фиксация, root cause, действия и сроки (SEV-1/2 — обязательный разбор); обновление базы знаний.
