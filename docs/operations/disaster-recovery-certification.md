# Disaster Recovery Certification — ЕАСУР

- **Status:** APPROVED · Stage 10 · Реализует подэтап 10.6, Business Continuity Validation Standard.
  Опирается на [Disaster Recovery Plan](../platform/disaster-recovery-plan.md) и
  [Backup Policy](../platform/backup-policy.md) (Stage 3) — подтверждает их исполнимость.

## 1. Назначение
Подтвердить готовность к аварийному восстановлению и непрерывности деятельности перед go-live:
исполнимость DR Plan, достижимость RTO/RPO, целостность данных при восстановлении.

## 2. Целевые показатели
- **RTO/RPO** для критического пути (приём/диспетчеризация) — заданы и подтверждаются испытаниями.
- **Доступность (SLO)** и сценарии деградации — определены.

## 3. Обязательные испытания (drills)
| Сценарий | Проверка | Критерий |
|----------|----------|----------|
| Восстановление из бэкапа | Целостность данных, время | ≤ RPO/RTO |
| Переключение на DR-контур | Работоспособность, согласованность | Функционирует |
| Отказ узла/зоны (HA) | Обслуживание оставшимися узлами | Без потери критпути |
| Отказ зависимости | Деградация/circuit breaker | Контролируемо |
| Возврат с DR (failback) | Согласованность после возврата | Данные целостны |

## 4. Результаты (заполняется при проведении)
Фактические RTO/RPO, отклонения, целостность, действия по устранению. Блокирующие отклонения —
до go-live.

## 5. Решение о сертификации
Certified / Certified с условиями / Not certified. Условие go-live: RTO/RPO достигнуты, данные целостны,
процедуры исполнимы персоналом по runbooks.

## 6. Периодичность
Повторные учения — регулярно (Long-Term Support / Maintenance Window Standards) и при значимых изменениях.
