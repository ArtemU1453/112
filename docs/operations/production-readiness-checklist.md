# Production Readiness Checklist — ЕАСУР

- **Status:** APPROVED · Stage 10 · Реализует Production Readiness Standard. Заполняется перед ORR и
  каждым рубежом go-live; блокирующие пункты не допускают продвижения.

## 1. Функциональная готовность
- [ ] Верификация системы против архитектуры/контрактов пройдена (10.1).
- [ ] Полная программа испытаний выполнена (unit/integration/system/E2E/UAT), 10.2.
- [ ] Критических дефектов нет; известные дефекты — с планом (не блокирующие).

## 2. Нефункциональная готовность
- [ ] Сертификация производительности пройдена (нагрузка/стресс/выносливость), 10.4.
- [ ] Сертификация безопасности пройдена (Zero Trust, скан/пентест), 10.5.
- [ ] Сертификация DR пройдена (RTO/RPO подтверждены), 10.6.

## 3. Эксплуатационная готовность
- [ ] Наблюдаемость: метрики/логи/трассы/дашборды/алерты настроены (Stage 3).
- [ ] Runbooks актуальны (Production Runbook, платформенные runbooks).
- [ ] Дежурство и эскалация определены (Incident Escalation Standard).
- [ ] Окна обслуживания и SLA/SLO согласованы.

## 4. Данные
- [ ] Миграции применяются идемпотентно и обратимо (Liquibase).
- [ ] Резервное копирование/восстановление проверено (Backup Policy).
- [ ] Целостность, ретенция и приватность данных обеспечены.

## 5. Безопасность
- [ ] Секреты в Vault (ADR-011); секретов в коде нет.
- [ ] mTLS/подпись/PKI интеграций активны (ADR-023); механизмы безопасности не отключены.
- [ ] Мониторинг безопасности и аудит включены.

## 6. Организационная готовность
- [ ] Персонал эксплуатации/поддержки обучен (Training Handbook).
- [ ] Поддержка (L1–L3) развёрнута (Production Support Standard).
- [ ] Пользователи/диспетчеры обучены; ИИ — только совещательный (AI Assistance Policy).

## Итог
Все блокирующие пункты закрыты → допуск к ORR/рубежу. Результат фиксируется в Operational Readiness
Review и итоговом Production Readiness Report.
