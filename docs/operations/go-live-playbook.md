# Go-Live Playbook — ЕАСУР

- **Status:** APPROVED · Stage 10 · Реализует National Deployment Standard, Deployment Program,
  ADR-024 (подэтап 10.7, 10.10). Операционный сценарий ввода в эксплуатацию по рубежам.

## 1. Роли и решение
Координатор развёртывания (ведёт сценарий), комиссия приёмки (go/no-go на рубеже), владельцы сервисов,
эксплуатация/поддержка, безопасность, DR-координатор, связь с заинтересованными сторонами.

## 2. Предстартовый чек (T-1)
- [ ] Release Acceptance пройден; артефакты неизменяемы; версии зафиксированы (TVMS).
- [ ] Production Readiness Checklist закрыт; ORR = Go для рубежа.
- [ ] Резервная копия перед выкатом снята и проверена.
- [ ] План отката подтверждён; DR-контур доступен; окно/уведомления согласованы.

## 3. Выкат (T0)
1. Развёртывание Helm-чарта `helm/emergency-112` (rolling update, values рубежа).
2. Применение миграций (идемпотентно, обратимо).
3. Прогрев/проверка зависимостей (БД, брокер, кэш, идентификация, интеграции).
4. Post-deployment validation (smoke/health/готовность) — [Guide](post-deployment-validation-guide.md).
5. Включение трафика на рубеже (пилот/регион/национальный).

## 4. Наблюдение (T0…T+)
Мониторинг SLA/SLO, ошибок, латентности, очередей, интеграций; дежурство усилено; статус-обновления
с фиксированным интервалом.

## 5. Критерии успеха рубежа
Целевые SLA/SLO достигнуты; SEV-1/SEV-2 отсутствуют; данные целостны; поддержка справляется → решение
о продвижении на следующий рубеж.

## 6. Критерии и процедура отката
Триггеры: критический дефект, недостижение SLA, потеря целостности, сбой безопасности. Действия:
остановить продвижение → `helm rollback` на предыдущую версию → при необходимости восстановление
данных (Backup/DR) → возврат к предыдущему рубежу → разбор (Incident Escalation).

## 7. Пост-go-live
Стабилизация, передача в эксплуатацию (Operations Handover), закрытие рубежа, обновление отчётов.
