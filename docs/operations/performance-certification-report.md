## 1. Идентификация
Версия/релиз · дата · исполнитель · окружение (идентично prod по EGS) · целевые показатели.

## 2. Целевые показатели (NFR)
- Пропускная способность критического пути (приём вызова → регистрация → диспетчеризация).
- Латентность (p50/p95/p99) по ключевым операциям и API.
- Ёмкость (одновременные пользователи/вызовы/события); поведение под пиком.
- Масштабирование (HPA) и деградация; отсутствие деградации целостности.

## 3. Виды испытаний
- **Нагрузочное** (целевая нагрузка), **стресс** (сверх пика/точка отказа), **выносливость**
  (длительная нагрузка, утечки), **всплеск** (spike), **объёмное** (данные).
