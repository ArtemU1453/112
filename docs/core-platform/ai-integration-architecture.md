# AI Integration Architecture — ЕАСУР

- **Status:** APPROVED · Stage 4 · Подчиняется AI Provider Abstraction Standard, TAP.

## Назначение
Универсальная интеграционная платформа для ИИ (AI Integration Framework). Возможности:
**STT, TTS, LLM, OCR, классификация, извлечение сущностей (NER), анализ документов**
(`AiCapability`). **Привязка к конкретной модели/поставщику запрещена** — все интеграции через
адаптеры (порт `AiProvider`, `platform-abstractions.ai`; `CAP-AI-001`).

## Модель абстракции
- `AiProvider.capabilities()` — какие возможности поддерживает адаптер.
- `AiRequest` / `AiResponse` — модель-агностичные запрос/ответ (вход, опции, результат, уверенность).
- Выбор адаптера — по требуемой `AiCapability`; несколько провайдеров сосуществуют.

## Границы этапа
На Stage 4 создаётся **только каркас интеграции** (порты/абстракции/стандарт). Реализация
ИИ-анализа вызовов/инцидентов — **запрещена** (Stage 4) и выполняется на профильном этапе через
адаптеры. Существующий ai-service (Stage 0) рассматривается как один из адаптеров возможностей.

## Провайдер-абстракция и суверенность
Провайдеры — self-hosted/локальные (AGSDS), заменяемы без изменения потребителей (AI Provider
Abstraction Standard). Выбор конкретной модели фиксируется ADR при внедрении.

## Соответствие
AI Provider Abstraction Standard, Integration Governance Standard, Security Architecture, TAP.
Без прикладной ИИ-логики анализа вызовов (Stage 4).
