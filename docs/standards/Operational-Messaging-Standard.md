# Operational Messaging Standard — ЕАСУР

- **Status:** APPROVED · Stage 7 · Согласован с Notification Platform (Stage 4), Messaging Architecture.

## Операционные сообщения (7.6)
Индивидуальные; групповые; системные уведомления. Атрибуты: **приоритет** сообщения; **подтверждение
получения** (acknowledgement); **журнал доставки**.

## Правила
- Сообщения работают offline-first (outbox); доставка гарантируется при связи (at-least-once,
  идемпотентность).
- Приоритетные сообщения (тревога/приказ) — первоочередная доставка и заметная индикация.
- Подтверждение получения фиксируется; журнал доставки доступен отправителю и в аудите.
- Единый контракт (`operational-messaging-api.yaml`) поверх платформенных уведомлений/шины
  (Notification Platform, ADR-008); не app-специфичный.

## Definition of Done
Определены индивидуальные/групповые/системные сообщения, приоритет, подтверждение, журнал доставки.
