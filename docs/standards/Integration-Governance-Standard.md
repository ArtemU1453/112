# Integration Governance Standard — ЕАСУР

- **Status:** APPROVED · Stage 4 · Согласован с Integration Architecture, EAGF.

## Цель
Определить правила заведения, эксплуатации и сопровождения интеграций (REST/gRPC/Message Bus/
файловый/пакетный обмен).

## Правила заведения интеграции
Каждая интеграция имеет: контракт (`contracts/`, API First), владельца, тип соединителя
(`ConnectorType`), модель данных (каноническая, CDM Standard), требования безопасности (RBAC/секреты),
мониторинг и обработку ошибок. Внешние интеграции — через RFC/ADR.

## Транспорты
- **REST/gRPC** — синхронные контракты.
- **Message Bus (Kafka)** — асинхронные события/команды (предпочтительно; Event Governance Standard).
- **Файловый/пакетный** — офлайн/внешние системы (в т. ч. air-gapped, AGSDS).

## Надёжность и безопасность
Идемпотентность, Retry, DLQ (Messaging); TLS/аутентификация; секреты — Vault (ADR-011); аудит
интеграционных вызовов. Ограничение доступа по сетевым политикам (Network Architecture).

## Наблюдаемость
Метрики/логи/трассы интеграций; алерты на ошибки/латентность (Operational Telemetry Standard).

## Управление изменениями
Изменения контрактов — API Lifecycle Management Standard; несовместимые — версия + миграция.

## Definition of Done
Определены правила заведения, транспорты, надёжность, безопасность и наблюдаемость интеграций.
