# Simulation Governance Standard — ЕАСУР

- **Status:** APPROVED · Stage 12 · Согласован с ADR-026, ADR-025 (ARB), Digital Twin Governance
  Standard, Simulation Validation Standard, Simulation Security Standard.

## Назначение
Управление платформой моделирования на всём жизненном цикле (12.3, 12.10): порядок запуска
экспериментов, изоляция, воспроизводимость, хранение результатов, перенос выводов.

## Принципы управления
- **Изоляция:** моделирование — только в изолированном контуре (ADR-026); влияние на прод исключено.
- **Воспроизводимость:** каждый запуск фиксирует версию сценария, модели двойника, параметры и seed
  (Simulation Validation Standard).
- **Прослеживаемость:** результаты сохраняются с полной привязкой к входным данным (Simulation Analytics).
- **Данные:** синтетические/обезличенные (Synthetic Data Standard); запрет использования их как реальных.
- **Перенос в прод:** только через утверждённую процедуру (RFC → ARB → ADR); автоперенос запрещён.

## Жизненный цикл эксперимента
Замысел → выбор/версия сценария и модели двойника → конфигурация параметров → запуск (одиночный/серийный)
→ сбор результатов → анализ (Analytics) → выводы → при необходимости — заявка на внедрение (Experiment
Approval / RFC→ARB→ADR).

## Роли
Владелец эксперимента, методолог (валидация), ARB (утверждение переноса), администратор контура (изоляция/
безопасность).

## Definition of Done
Определены принципы, жизненный цикл, роли и правила переноса результатов моделирования.
