# Technical Debt Governance Standard — ЕАСУР

- **Status:** APPROVED · Stage 11 · Согласован с ADR-025, Quality Gates, Architecture Evolution
  Standard, Continuous Improvement Framework.

## Назначение
Управление техническим долгом на всём жизненном цикле (11.3). Накопление долга без регистрации
запрещено (запрет этапа).

## Жизненный цикл долга
1. **Регистрация** — в Technical Debt Register (идентификатор, источник, дата, владелец).
2. **Классификация** — код/архитектура/тесты/инфраструктура/документация/безопасность/данные.
3. **Оценка влияния** — риск, стоимость обслуживания, влияние на надёжность/безопасность/скорость.
4. **Приоритизация** — по влиянию и срочности (критический/высокий/средний/низкий).
5. **План устранения** — действие, срок, ответственный, связь с релизом/RFC (при архитектуре).
6. **Контроль выполнения** — статус, метрики тренда долга, отчётность ARB.

## Правила
- Каждый значимый компромисс фиксируется как запись долга при внесении.
- Критический долг (безопасность/надёжность критпути) — блокирует накопление, планируется немедленно.
- Архитектурный долг устраняется через RFC→ARB→ADR.
- Тренд долга — метрика непрерывного совершенствования (Operational Improvement).

## Definition of Done
Определены жизненный цикл, классификация, приоритизация и контроль технического долга; ведётся реестр.
