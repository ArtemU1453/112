# API Lifecycle Management Standard — ЕАСУР

- **Status:** APPROVED · Stage 4 · Согласован с API Governance Standard (AGS), EAGF.

## Цель
Определить жизненный цикл API от контракта до вывода из эксплуатации (API First).

## Жизненный цикл
```
Design (контракт) → Review → Published → Deprecated → Retired
```
Контракт (OpenAPI, `contracts/`) утверждается **до** реализации (Constitution ARTICLE 12).

## Версионирование
- Версия в пути (`/v1`); аддитивные изменения — без смены версии; несовместимые — новая мажорная.
- Одновременно поддерживается ограниченное число мажорных версий; политика депрекации объявляется
  заранее (срок, замена).

## Совместимость
Изменения контракта проходят проверку обратной совместимости (Compatibility Check, RFC при
несовместимости). Контракт — источник истины для клиентов.

## Категории API
Internal / Admin / External (`contracts/{internal,admin,external}`) с разными политиками доступа и
темпами изменений; внешние — через Integration Governance Standard + RFC/ADR.

## Управление
Каждый API имеет владельца, контракт, тесты контракта и мониторинг использования. Retired-версии
удаляются после переходного периода.

## Definition of Done
Определён жизненный цикл, версионирование, совместимость и политика депрекации; все API имеют
контракты до реализации.
