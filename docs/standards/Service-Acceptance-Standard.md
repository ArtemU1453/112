# Service Acceptance Standard — ЕАСУР

- **Status:** APPROVED · Stage 10 · Согласован с Service Maturity Model, Operational Acceptance
  Standard, Production Readiness Standard.

## Назначение
Критерии приёмки отдельного сервиса в эксплуатацию (10.3, 10.8). Дополняет операционную приёмку на
уровне каждого сервиса до уровня зрелости, требуемого для продуктива.

## Критерии приёмки сервиса
- Достигнут требуемый уровень **Service Maturity Model** (документация, тесты, наблюдаемость, SLO).
- Health/readiness/liveness-пробы, метрики, логи, трассы, алерты — настроены.
- Runbook сервиса, зависимости, деградация и восстановление — описаны.
- Контракты (API/события) опубликованы и версионируются; обратная совместимость обеспечена.
- Безопасность: авторизация (RBAC/scope), секреты в Vault, аудит значимых действий.
- Резервное копирование/миграции данных сервиса — проверены.

## Процесс
Заявка владельца сервиса → проверка критериев → устранение замечаний → приёмка сервиса → включение
в эксплуатационный контур и мониторинг.

## Definition of Done
Определены критерии и процесс приёмки отдельного сервиса в промышленную эксплуатацию.
