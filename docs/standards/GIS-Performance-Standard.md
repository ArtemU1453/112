# GIS Performance Standard — ЕАСУР

- **Status:** APPROVED · Stage 6 · Согласован с GIS Architecture, Operational Telemetry Standard.

## Требования
- Пространственные индексы: **GiST** для геометрий, `pg_trgm` для адресного поиска (ADR-003).
- Целевые ориентиры (SLO): геокодирование/поиск ближайших станций — интерактивная латентность
  (доли секунды); расчёт маршрута — секунды; отрисовка оперативной карты — плавно при типовой
  плотности.
- Кэширование частых запросов (Redis, ADR-003) и тайлов; пагинация пространственного поиска.
- Тайлы — предгенерация/кэш (self-hosted, ADR-015).

## Контроль
Метрики латентности/нагрузки гео-запросов — Prometheus (Operational Telemetry Standard); алерты при
деградации; нагрузочное тестирование ключевых сценариев (Performance-окружение, EGS).

## Definition of Done
Определены индексация, целевые SLO, кэширование и мониторинг производительности GIS.
