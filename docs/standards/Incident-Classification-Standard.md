# Incident Classification Standard — ЕАСУР

- **Status:** APPROVED · Stage 5 · Согласован с Emergency Data Dictionary, Operational Decision
  Support Standard. Синхронизирован с `IncidentType`/`IncidentPriority`.

## Классификация (тип)
Тип происшествия — `IncidentType` (Emergency Data Dictionary §1). Классификация выполняется
диспетчером; ИИ может предлагать вариант (AI Assistance Policy).

## Приоритизация
Приоритет — `IncidentPriority`: CRITICAL, HIGH, MEDIUM, LOW. Ориентиры (регламент):
- **CRITICAL** — угроза жизни/массовые последствия (пожар с людьми, взрыв, ЧС, множественные
  пострадавшие).
- **HIGH** — прямая угроза (активный пожар, ДТП с пострадавшими, утечка газа).
- **MEDIUM** — потенциальная угроза без немедленной опасности жизни.
- **LOW** — справочные/несрочные обращения, ложные вызовы.

## Обязательные данные по типу (пример)
Для полноты классификации требуются подтверждённые сущности (Decision Support проверяет наличие):
- Все типы: адрес/координаты, тип происшествия.
- FIRE/HAZMAT: наличие людей, опасные вещества.
- MEDICAL: число пострадавших.
- TRAFFIC_ACCIDENT: транспорт, пострадавшие.

## Регламенты
Классификация и последующие действия — в пределах утверждённых регламентов; выход за регламент
запрещён. Изменение правил классификации — через RFC/ADR с обновлением стандарта.

## Definition of Done
Определены типы/приоритеты/обязательные данные/регламенты; предложения ИИ подтверждаются диспетчером.
