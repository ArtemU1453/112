# Offline Synchronization Standard — ЕАСУР

- **Status:** APPROVED · Stage 7 · ADR-017 · Согласован с Synchronization Architecture.

## Требования (7.4/7.5)
- **Полная работа без сети:** чтение/запись на локальном шифрованном хранилище.
- **Очередь синхронизации (outbox):** изменения накапливаются и передаются при связи; сохраняется
  между сессиями.
- **Двусторонняя дельта-синхронизация:** pull/push только изменений; частичная синхронизация по
  областям/приоритетам; пакетная передача.
- **Контроль версий и целостности:** версии сущностей, контрольные суммы; повторная передача при сбое.
- **Разрешение конфликтов:** версия/время + доменные правила; критичные — вручную (без автозатирания).
- **Восстановление после потери связи:** экспоненциальные повторы, идемпотентность на сервере.
- **Журнал синхронизации:** направление/объём/результат/конфликты/ошибки.

## Инварианты
Отсутствие потери данных; устойчивость к обрывам; отсутствие дублей (идемпотентность). Синхронизация
— через общий контракт (`mobile-sync-api.yaml`), не app-специфичный.

## Definition of Done
Определены offline-работа/outbox/дельта/версии/целостность/конфликты/восстановление/журнал.
