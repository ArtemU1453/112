# API Versioning Standard — ЕАСУР

- **Status:** APPROVED · Stage 9 · Согласован с API Lifecycle Management Standard (Stage 4),
  Contract Governance Standard.

## Правила версионирования
- Версия в пути (`/v1`) или в схеме/пакете (gRPC); SemVer для контрактов.
- **Аддитивные** изменения (новые необязательные поля/операции) — без смены мажорной версии.
- **Несовместимые** изменения — новая мажорная версия; старая поддерживается до согласованного
  вывода (депрекация с уведомлением и сроком, Integration SLA Standard).
- Клиенты объявляют используемую версию; платформа маршрутизирует по версии.
- Число одновременно поддерживаемых мажорных версий ограничено политикой.

## Обязательность
**Все API версионируются** (обязательная проверка этапа). Изменение без версии/плана совместимости
запрещено.

## Definition of Done
Определены схема версий, правила совместимости, депрекация и обязательность версионирования.
