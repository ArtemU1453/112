# Definition of Ready (DoR) — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0

Задача может быть взята в работу (переход `Backlog → Ready`) только если выполнены все условия.

## Критерии готовности

- [ ] Задача имеет уникальный идентификатор и ссылку на Epic/Feature (Backlog Structure).
- [ ] Сформулированы бизнес-ценность и техническое описание.
- [ ] Определены критерии приёмки (acceptance criteria), измеримые и проверяемые.
- [ ] Указаны зависимости (задачи, сервисы, контракты) и они не блокируют старт.
- [ ] Определён приоритет и оценка сложности.
- [ ] Соответствие архитектуре подтверждено: изменение укладывается в утверждённые ADR/Baseline,
      либо заведён RFC (для архитектурно значимых изменений).
- [ ] Определены контракты (API First): при изменении API — черновик OpenAPI; при изменении
      данных (Database First) — план миграции.
- [ ] Определён требуемый уровень тестирования (Unit/Integration/Contract/E2E/Performance).
- [ ] Определены влияние на безопасность (роли/RBAC/ПДн) и на наблюдаемость.
- [ ] Понятно, какие документы потребуют обновления (Document Synchronization).

Если хотя бы один пункт не выполнен, задача остаётся в `Backlog`.
