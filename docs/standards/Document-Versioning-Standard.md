# Document Versioning Standard — ЕАСУР

- **Status:** APPROVED · Stage 13 · Согласован с ADR-028, Regulatory Document Standard, Knowledge
  Lifecycle Standard, Git Strategy, API Versioning Standard (аналогия подхода).

## Назначение
Правила версионирования документов и единиц знаний (13.2, 13.3): схема версий, неизменяемость истории,
связь версий с утверждением и публикацией.

## Схема версий
- Семантическая версия документа `MAJOR.MINOR.PATCH`:
  - **MAJOR** — существенное изменение содержания/требований (переознакомление обязательно);
  - **MINOR** — дополнения/уточнения без изменения обязательных требований;
  - **PATCH** — редакционные правки (опечатки/форматирование).
- Каждая версия фиксирует: автора, дату, статус, орган утверждения, основание, дифф к предыдущей.

## Неизменяемость истории
- История версий **неизменяема**; удаление версий/истории запрещено (запрет этапа).
- Замена — публикацией новой версии; прошлые переводятся в Superseded (не удаляются).
- Ссылки (правила/SOP/обучение) указывают конкретную версию для трассируемости.

## Правила
- В работе применяется только текущая Published-версия; черновики недоступны для применения.
- MAJOR-изменение инициирует обязательное ознакомление (Knowledge Distribution).
- Восстановление предыдущей редакции — публикацией версии, а не правкой истории.

## Definition of Done
Определены схема версий, неизменяемость истории и правила применения версий документов.
