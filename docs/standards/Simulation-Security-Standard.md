# Simulation Security Standard — ЕАСУР

- **Status:** APPROVED · Stage 12 · Согласован с ADR-026, Security Architecture (Stage 4), Zero Trust
  (ADR-023), Synthetic Data Standard, Air-Gapped & Sovereign Deployment Standard.

## Назначение
Требования безопасности и изоляции платформы цифрового двойника и моделирования (12.1–12.10).
Гарантирует отсутствие влияния на промышленную эксплуатацию.

## Изоляция
- **Отдельный контур:** двойник/симуляция развёртываются в изолированной среде; сетевого пути к прод,
  позволяющего влиять на него, нет (нет обратной записи).
- **Однонаправленный импорт:** только офлайн-снимок справочной модели из прод; онлайн-интеграции с прод
  запрещены.
- **Разделение данных:** учебные/синтетические данные изолированы от реальных (Exercise Data Governance).

## Контроль доступа и защита
- Аутентификация/авторизация по ролям (OIDC/RBAC, ADR-007); least privilege; аудит доступа.
- Секреты — в защищённом хранилище (ADR-011); секретов в коде нет; шифрование данных при передаче/хранении.
- Zero Trust внутри контура; сегментация; мониторинг безопасности.

## Суверенность
Self-hosted/air-gapped (AGSDS); без вендор-лока (TAP); данные не покидают контролируемый периметр.

## Инварианты (обязательные проверки этапа)
Двойник изолирован от прод; экспериментальные модели ИИ не влияют на рабочую систему; синтетические
данные не смешиваются с реальными.

## Definition of Done
Определены изоляция, однонаправленный импорт, контроль доступа, защита данных и суверенность контура моделирования.
