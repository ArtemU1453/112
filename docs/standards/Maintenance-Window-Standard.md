# Maintenance Window Standard — ЕАСУР

- **Status:** APPROVED · Stage 10 · Согласован с Production Support Standard, Release Governance
  Standard, Business Continuity Validation Standard.

## Назначение
Правила планирования и проведения окон технического обслуживания для системы жизнеобеспечения,
работающей 24/7 (10.8). Обслуживание не должно нарушать доступность критического пути.

## Принципы
- **Приоритет — непрерывность:** обновления выполняются без простоя, где возможно (rolling update,
  обратно совместимые миграции, поэтапно).
- **Окна** согласуются заранее, планируются на периоды низкой нагрузки, уведомляются заинтересованные
  стороны; аварийные окна — по процедуре инцидентов.
- **Резервирование:** при обслуживании узла нагрузка обслуживается остальными (HA); DR-контур доступен.
- **Обратимость:** любой шаг обслуживания имеет план отката; проверка работоспособности после.

## Классы работ
Плановые (обновления/патчи/оптимизация), срочные (уязвимости безопасности), аварийные (устранение
сбоя). Каждый класс — свои сроки согласования и уведомления.

## Аудит
Окна, работы и результаты фиксируются; связь с релизами/RFC — сохраняется.

## Definition of Done
Определены принципы, классы, согласование и обратимость окон обслуживания для режима 24/7.
