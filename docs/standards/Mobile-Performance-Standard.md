# Mobile Performance Standard — ЕАСУР

- **Status:** APPROVED · Stage 7 · Согласован с Mobile Architecture, GIS Performance Standard.

## Требования
- Отзывчивость UI: плавная прокрутка списков/карты при типовом объёме данных; отсутствие блокировки
  главного потока (тяжёлые операции — фоново).
- Локальные запросы (offline-хранилище) — интерактивная латентность; индексы для частых выборок.
- Карта: кэш тайлов (self-hosted, ADR-015); ограничение числа объектов на экране (кластеризация).
- Синхронизация — фоновая, не блокирует работу; дельта/батчинг под слабый канал.
- Медиа — прогрессивная/фоновая загрузка; сжатие перед отправкой.

## Контроль
Метрики производительности (старт, кадры, использование памяти) собираются (Operational Telemetry);
целевые ориентиры проверяются на целевых устройствах.

## Definition of Done
Определены отзывчивость/латентность/карта/синхронизация/медиа и контроль производительности.
