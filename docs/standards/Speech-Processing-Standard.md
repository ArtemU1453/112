# Speech Processing Standard — ЕАСУР

- **Status:** APPROVED · Stage 5 · Согласован с AI Integration Architecture, AI Provider Abstraction
  Standard, AI Assistance Policy. Реализуется ai-service через адаптеры.

## Возможности (5.3)
Через порт `AiProvider` (CAP-AI-001), модель-агностично:
- **Speech-to-Text** (STT);
- **определение языка** транскрипта;
- **определение говорящего** (диаризация) — если включено архитектурой;
- **разбиение по репликам** (сегменты) с **временными метками**;
- **уверенность распознавания** (confidence 0..1) на сегмент/транскрипт;
- **повторная обработка** записи (переобработка при новой модели/параметрах).

## Модель данных
Транскрипт = список сегментов (`text`, `speaker?`, `startMs`, `endMs`, `language`, `confidence`)
— Emergency Data Dictionary §5. Транскрипт — исходный материал для извлечения сущностей (5.4).

## Абстракция и суверенность
Конкретная модель/провайдер STT не фиксируется (AI Provider Abstraction Standard); провайдеры
self-hosted/локальные (AGSDS). Замена провайдера — без изменения потребителей.

## Политика ИИ
Результаты STT — исходные данные, не решения; извлечённые из них сущности требуют подтверждения
диспетчером (AI Assistance Policy). Все вызовы обработки речи журналируются (Operational Audit).

## Качество
Пороги уверенности и деградация (ручной ввод при низком качестве/сбое) — Operational Decision
Support Standard. Метрики качества — Operational Telemetry Standard.

## Definition of Done
Определены STT/язык/говорящий/реплики/метки/уверенность/повторная обработка через абстракции;
согласовано с политикой ИИ.
