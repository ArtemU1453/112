# Knowledge Retention Standard — ЕАСУР

- **Status:** APPROVED · Stage 13 · Согласован с ADR-028, Knowledge Lifecycle Standard, Document
  Versioning Standard, Data Retention for Analytics Standard (Stage 8), Cross-System Audit Standard.

## Назначение
Правила хранения и удержания знаний и истории изменений (13.2): сроки, архивирование, неизменяемость
истории, доступность для аудита и трассируемости.

## Принципы
- **Неудаление истории:** история версий/статусов/ознакомления/публикаций **не удаляется** (запрет
  этапа); устаревшее переводится в статус (Superseded/Expired/Archived), а не стирается.
- **Архивирование:** выведенные из применения единицы сохраняются в архиве с сохранением связей и версий.
- **Сроки удержания:** определяются по типу знаний и нормативным требованиям; удержание не короче
  установленного регламентом.
- **Доступность для аудита:** история и основания доступны для проверки (полная трассируемость, EAGF).

## Хранение и доступ
Версионируемое хранилище знаний; резервное копирование (Backup Policy); доступ по ролям; аудит доступа.
Экспорт/архив — с сохранением метаданных (версия/статус/основание).

## Правила
- Любое «удаление» реализуется как перевод в архив/статус, а не физическое стирание истории.
- Восстановление — из архива/версии; целостность истории сохраняется.

## Definition of Done
Определены неудаление истории, архивирование, сроки удержания и доступность знаний для аудита.
