# Coding Standards — ЕАСУР

> Приоритет SSOT: 9. Статус: APPROVED · Baseline: v1.0

Стандарты кодирования по языкам/форматам. Проверяются статическим анализом и Code Review.

## Общие правила
- Единый формат ошибок REST — RFC 7807 (`application/problem+json`): `type`, `title`, `status`,
  `detail`, `instance`; для валидации — перечень нарушений полей.
- Структурированное логирование (JSON), без ПДн и секретов в логах; корреляция запросов.
- Обработка ошибок явная; исключения — доменные типы, транслируемые в коды HTTP централизованно
  (`@RestControllerAdvice`).
- Никаких TODO/заглушек/мёртвого кода в основной ветке (Constitution, ARTICLE 3).

## Java
- Версии Java (LTS) и Spring Boot — по каталогу версий (`config/toolchain/versions.yaml`, TVMS).
  Форматирование — Google Java Format / IDE-конвенции проекта.
- Иммутабельность: DTO — `record`; сущности — инкапсуляция инвариантов в домене (без анемичных
  сеттеров для инвариантных полей).
- Слои: `domain` не зависит от Spring/JPA-специфики там, где возможно; бизнес-правила — в домене.
- Транзакции: `@Transactional` на уровне application-сервисов; чтения — `readOnly = true`.
- Валидация: Bean Validation (`jakarta.validation`) на входных DTO.
- Персистентность: JPA/Hibernate; схема — Liquibase; `ddl-auto: validate`. Оптимистическая
  блокировка `@Version` для конкурентно изменяемых агрегатов.
- Kafka: продюсеры идемпотентны (`acks=all`); консьюмеры идемпотентны и толерантны к неизвестным
  полям; группы по имени сервиса.
- Null-безопасность: избегать `null` в API домена; `Optional` в репозиториях.
- Тесты: JUnit 5, AssertJ, Mockito; интеграционные — Testcontainers (PostgreSQL/Kafka).

## TypeScript / React
- `strict: true`; без `any` без обоснования; типизировать контракты API.
- Функциональные компоненты + хуки; состояние — Redux Toolkit (срезы), без прямых мутаций.
- Побочные эффекты — в thunks/эффектах; сеть — через единый API-клиент с инъекцией JWT.
- Компоненты презентационные/контейнерные разделены; доступность (a11y) для ключевых элементов.
- Форматирование/линт — ESLint + Prettier; сборка — Vite; проверка типов `tsc --noEmit` в CI.

## Python (ai-service)
- Python (версия — по каталогу версий); FastAPI; типизация (type hints) обязательна; `from __future__ import annotations`.
- Пакетная структура: `app/{routers,services,models,config}`; бизнес-логика — в `services`.
- Валидация — Pydantic-модели; ошибки — HTTP-исключения FastAPI.
- Тесты — pytest; конфигурация — через переменные окружения (`config.py`).
- Стиль — ruff/flake8 + black-совместимое форматирование.

## SQL
- Только через Liquibase changeSet'ы; один changeSet — атомарное изменение с `rollback`.
- Явные ограничения (NOT NULL, FK, UNIQUE, CHECK); индексы под запросы; именование — DGS.
- Никаких деструктивных изменений без стратегии «расширить → мигрировать → сжать».

## YAML
- Отступ 2 пробела; без табов; булевы `true/false`; секреты — не хранить в git (плейсхолдеры).
- `application.yml`: значения через `${ENV:default}`; профили окружений.

## Markdown
- Заголовки по иерархии; таблицы для сравнений; ссылки относительные внутри `docs/`.
- Документы Baseline содержат статус/версию в шапке.

## Dockerfile
- Многостадийная сборка; минимальный runtime-образ; non-root пользователь где возможно.
- Явные версии базовых образов; `.dockerignore`; кэшируемые слои зависимостей;
  `HEALTHCHECK`/пробы согласованы с `management`/`/health`.
