# Mobile Lifecycle Management Standard — ЕАСУР

- **Status:** APPROVED · Stage 7 · Согласован с Release Governance Standard, Device Security Standard.

## Жизненный цикл приложений
```
Build → Sign → Distribute (MDM) → Install → Update → Deprecate → Retire
```
- Сборка/подпись артефактов (целостность, Release Governance Standard); версии — SemVer.
- Распространение и обновление — через MDM/внутренний канал (ADR-018), не обязательно публичные сторы.
- Совместимость клиент↔сервер: поддерживаются N последних мажорных версий контрактов (API Lifecycle);
  принудительное обновление при несовместимости.
- Депрекация/вывод версий — с переходным периодом; данные пользователя не теряются (миграция/синхронизация).

## Правила
Единый SDK/дизайн-система обновляются согласованно; изменения — с обратной совместимостью или
мажорной версией + миграцией.

## Definition of Done
Определены сборка/подпись/дистрибуция/обновление/совместимость/депрекация мобильных приложений.
