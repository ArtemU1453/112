# Domain Modeling Standard (DMS) — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0

Единые правила моделирования предметной области (DDD). Обеспечивает единообразие модели во
всех сервисах.

## Строительные блоки

| Блок | Определение | Правила |
|------|-------------|---------|
| **Bounded Context** | Граница модели и языка (= сервис) | Своя схема данных; интеграция через контракты/события |
| **Aggregate** | Кластер объектов с корнем и границей согласованности | Изменяется только через корень; одна транзакция — один агрегат |
| **Aggregate Root** | Корень агрегата, точка входа инвариантов | Гарантирует инварианты; внешние ссылки — по идентификатору |
| **Entity** | Объект с идентичностью и жизненным циклом | Идентичность стабильна; равенство по id |
| **Value Object** | Неизменяемый объект-значение без идентичности | Иммутабелен; равенство по значению (координаты, адрес) |
| **Domain Event** | Факт, произошедший в домене | Прошедшее время; публикуется через Kafka по AGS/DGS |
| **Domain Service** | Доменная операция, не принадлежащая одной сущности | Без состояния; в доменном слое |
| **Repository** | Доступ к агрегатам | Один репозиторий на агрегат-корень |

## Правила

- Инварианты — внутри агрегата; недопустимые состояния невозможны по построению (пример:
  статусная модель Incident разрешает только допустимые переходы).
- Границы согласованности: одна транзакция изменяет один агрегат; межагрегатная/межсервисная
  согласованность — через доменные события (eventual consistency).
- Ссылки между агрегатами/контекстами — по идентификатору, не по объектной навигации.
- Value Object предпочтительнее примитивов для доменных значений (координаты, приоритет, статус).
- Доменные события именуются в прошедшем времени и соответствуют топикам Kafka (ADR-008).

## Ubiquitous Language (глоссарий, извлекается из реализации)

| Термин | Контекст | Значение |
|--------|----------|----------|
| Происшествие (Incident) | incident | Карточка события, требующего реагирования |
| Приоритет (Priority) | incident | CRITICAL/HIGH/MEDIUM/LOW |
| Статус происшествия | incident | RECEIVED→CLASSIFIED→DISPATCHED→IN_PROGRESS→RESOLVED→CLOSED (+CANCELLED) |
| Подразделение/Наряд (Unit) | dispatch | Оперативная единица (расчёт/экипаж) |
| Назначение (Assignment) | dispatch | Связь наряд↔происшествие |
| Вызов (Call) | telephony | Телефонное обращение и его жизненный цикл |
| Станция (Station) | gis | Пожарная часть/пост/подразделение на карте |
| Зона ответственности (Response Zone) | gis | Полигон покрытия |
| Событие аудита (Audit Event) | audit | Зафиксированное действие/изменение |

Глоссарий поддерживается синхронно с кодом; новые термины добавляются в рамках задачи,
вводящей их (Document Synchronization).

## Моделирование новых контекстов

Новый bounded context вводится только через ADR/RFC. Для него определяются: агрегаты и корни,
инварианты, доменные события, репозитории, язык предметной области, интеграция (контракты/топики).
