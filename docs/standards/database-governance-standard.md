# Database Governance Standard (DGS) — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0 · Приоритет SSOT: 8 (Database Schema)

Правила проектирования схемы, именования, миграций, индексов, ограничений, аудита,
архивирования и удаления данных (Database First, Constitution ARTICLE 13).

## Общие принципы

- **Database-per-service** (ADR-002/003): сервис владеет своей схемой; чужие таблицы недоступны.
- Изменения схемы — только через **Liquibase**-миграции; `ddl-auto: validate`.
- PostgreSQL + PostGIS для геоданных; JSONB — только для аудита/гетерогенных payload'ов.

## Именование

- Таблицы — `snake_case`, множественное число (`incidents`, `assignments`, `audit_events`).
- Столбцы — `snake_case`; PK — `id`; внешний ключ — `<entity>_id`.
- Временные метки — `created_at`, `updated_at` (UTC, `timestamptz`).
- Индексы — `ix_<table>_<cols>`; уникальные — `ux_<table>_<cols>`; PK — `pk_<table>`;
  FK — `fk_<table>_<ref>`; CHECK — `ck_<table>_<rule>`.

## Ключи и ограничения

- Первичные ключи — суррогатные (UUID/bigserial по контексту); естественные ключи — как UNIQUE.
- Явные ограничения целостности: NOT NULL, FK, UNIQUE, CHECK. Инварианты домена, выразимые в БД,
  дублируются ограничениями (defence in depth).
- Внешние ключи — в пределах одной схемы сервиса; межсервисные ссылки — только по идентификатору
  без физического FK (границы контекстов).

## Индексы

- Индексируются столбцы фильтрации/сортировки/соединения по реальным запросам.
- Геоиндексы (GiST) для PostGIS-геометрии; GIN — для JSONB (аудит) и `pg_trgm` (нечёткий поиск).
- Избыточные индексы не создаются (стоимость записи); решения обосновываются.

## Миграции

- Один changeSet — атомарное изменение с обязательным `rollback`.
- Стратегия несовместимых изменений — **expand → migrate → contract** (расширить схему,
  перенести данные, затем удалить старое) для сохранения обратной совместимости в переходный период.
- Начальные данные (seed) — отдельными changeSet'ами, идемпотентны.

## Аудит данных

- Значимые доменные действия публикуются событием и агрегируются в audit-service (JSONB + GIN).
- Оперативные таблицы содержат `created_at`/`updated_at`; для конкурентно изменяемых агрегатов —
  версия оптимистической блокировки.

## Хранение, архивирование, удаление

- Определяются сроки хранения для чувствительных данных (записи разговоров, ПДн) — открытый
  вопрос Vision §8, фиксируется отдельным решением/ADR.
- Архивирование/партиционирование крупных таблиц (аудит, вызовы) — при росте объёмов, через RFC.
- Удаление ПДн — по политике минимизации и требованиям ИБ; удаление аудита — контролируемо и
  журналируемо.

## Резервное копирование

- Регламент `pg_dump` всех БД (см. `infrastructure/backup/` и Helm CronJob); проверка
  восстановления — часть эксплуатационных процедур (operations.md).
