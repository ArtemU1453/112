# Document Lifecycle Management Standard — ЕАСУР

- **Status:** APPROVED · Stage 4 · Согласован с Document Platform, File Storage Abstraction.

## Цель
Определить жизненный цикл документов платформы: шаблоны, генерация, экспорт, печать, электронное
хранение.

## Жизненный цикл
```
Template → Generate → Export/Print → Store → Retain → Dispose
```
Генерация — через порт `DocumentRenderer` (форматы PDF/DOCX/HTML/CSV); хранение — File Storage
Abstraction (S3-совместимое, ADR-013); движок рендеринга не фиксируется (`CAP-DOC-001`, TAP).

## Шаблоны
Шаблоны версионируются; модель представления передаётся отдельно (без бизнес-логики в шаблоне).
Локализация — Localization Platform (языки/форматы даты/чисел).

## Хранение и удержание
Сроки хранения и удаление — по правовым требованиям и Backup Policy/EGS; целостность — контрольные
суммы (AGSDS). Доступ к документам — RBAC + аудит.

## Управление
Изменения шаблонов/форматов — через PR/ревью; новый формат/движок — при необходимости через ADR.

## Definition of Done
Определён жизненный цикл, форматы, хранение/удержание и локализация документов.
