# Mobile Security Standard — ЕАСУР

- **Status:** APPROVED · Stage 7 · Согласован с Security Architecture, Device Security Standard,
  ADR-018.

## Требования (безопасность приложения)
- **Шифрование локального хранилища** (SQLCipher/эквивалент, ADR-017).
- **Защищённое хранение ключей** (Android Keystore/Secure Enclave); ключи не в коде/файлах.
- **Аппаратные механизмы защиты** (если доступны): keystore, биометрия для разблокировки.
- **Проверка целостности приложения** и **обнаружение модификации** (tamper detection).
- **Авто-завершение сессии** по политике (таймаут/удалённый выход через MDM).
- **Журнал безопасности** (события входа/выхода/доступа/нарушений) — синхронизируется с аудитом.

## Данные
Чувствительные данные не передаются в открытом виде (TLS); минимизация ПДн (Geospatial/Privacy
стандарты). Секреты — запрещено хранить в коде приложения (запрет этапа).

## Аутентификация/авторизация
OIDC/JWT (Keycloak, ADR-007); RBAC-роли; учётные записи устройств. Управление устройствами/
сертификатами — MDM (ADR-018).

## Definition of Done
Определены шифрование хранилища/ключей, целостность/tamper, авто-выход, журнал безопасности, защита
передачи.
