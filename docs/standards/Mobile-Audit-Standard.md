# Mobile Audit Standard — ЕАСУР

- **Status:** APPROVED · Stage 7 · Согласован с Audit Platform (Stage 4), Operational Audit (Stage 5).

## Требования
- Журналируются: действия пользователя (статусы/формы/команды/медиа), события безопасности,
  синхронизация, доступ к данным.
- Мобильный аудит работает offline (локально) и **синхронизируется** с платформенным аудитом
  (неизменяемость на сервере, append-only).
- Записи содержат: кто/что/над чем/когда/устройство/correlationId; без утечки секретов/лишних ПДн.

## Правила
Все операции журналируются (обязательная проверка этапа); аудит нельзя обойти/удалить на клиенте
незаметно (защита целостности до синхронизации).

## Definition of Done
Определены состав аудита, offline-журнал и синхронизация с платформенным аудитом.
