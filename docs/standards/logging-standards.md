# Logging Standards — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0

Единый стандарт логирования для всех сервисов (Observability by Design, Constitution ARTICLE 11).

## Уровни логирования
| Уровень | Назначение |
|---------|-----------|
| ERROR | Сбой операции, требующий внимания; нарушение инварианта/недоступность зависимости |
| WARN | Аномалия без прерывания операции; деградация, повтор, приближение к лимиту |
| INFO | Значимые бизнес/жизненные события (старт/стоп, приём вызова, смена статуса) |
| DEBUG | Диагностическая информация для разработки (выключено в prod по умолчанию) |
| TRACE | Максимально детально; только для локальной отладки |

Уровень по окружениям задаётся конфигурацией (Environment Governance Standard), не хардкодится.

## Структурированные логи
- Формат — **JSON** (машиночитаемый), одна запись = один объект.
- Обязательные поля: `timestamp`, `level`, `service`, `logger`, `message`, `correlationId`,
  `traceId` (при наличии трассировки), `spanId`, `thread`.
- Бизнес-контекст — в структурированных полях (например, `incidentId`, `unitId`), не в тексте.

## Корреляционные идентификаторы
- `correlationId` присваивается на периметре (Gateway) и пробрасывается через HTTP-заголовок и
  заголовки сообщений Kafka по всей цепочке обработки.
- Обеспечивает сквозную прослеживаемость запроса/события между сервисами.

## Формат времени
- **UTC**, ISO-8601 с миллисекундами (`2026-07-21T12:34:56.789Z`).

## Маскирование чувствительных данных
- **Запрещено** логировать: пароли, токены, секреты, полные персональные данные заявителей,
  содержимое записей разговоров.
- Телефоны/адреса/ФИО — маскируются или логируются только идентификаторы сущностей.
- Маскирование выполняется до записи в лог (фильтры/сериализаторы).

## Правила
- Никаких `System.out`/`print` — только логгер (SLF4J для Java, стандартный logging для Python).
- Исключения логируются с контекстом и стеком на уровне ERROR однократно (без дублирования по слоям).
- Логи не заменяют метрики/трассировку — используются совместно.

## Хранение и доставка
- Логи пишутся в stdout/stderr контейнера; сбор и агрегация — платформенной возможностью
  (`CAP-OBS-002`, см. Technology Mapping). Ротация/хранение — по окружению (EGS).
