# SOP Lifecycle Standard — ЕАСУР

- **Status:** APPROVED · Stage 13 · Согласован с ADR-028, SOP Governance Guide, Document Versioning
  Standard, Incident Classification/Command Standards (Stage 5–6), Dispatcher Workflow Standard.

## Назначение
Жизненный цикл стандартных операционных процедур (13.3): создание, согласование, публикация, версии,
привязка к типам происшествий, обязательные контрольные точки.

## Жизненный цикл SOP
```
Draft → Review → Approved → Published → Maintained → Deprecated → Archived
```
- Публикуются и применяются только **Approved/Published** SOP; история версий неизменяема.
- Deprecated/Archived сохраняются (не удаляются); замена — новой версией.

## Обязательные элементы SOP
- Идентификатор, версия, статус, владелец, орган утверждения, основание (нормативные ссылки).
- **Привязка к типам происшествий** (IncidentType, Stage 5) и приоритетам.
- **Шаги процедуры** и **обязательные контрольные точки** (mandatory checkpoints), критерии выполнения.
- Роли-исполнители; связанные документы/правила.

## Применение и контроль
- SOP используются как основа рекомендаций слоя поддержки решений (13.7) — совещательно, с объяснением.
- Обязательные контрольные точки отслеживаются; пропуск фиксируется (аудит), но система **не изменяет
  рабочий процесс автоматически** без утверждения.
- Изменение SOP — только через утверждённый процесс согласования (Knowledge Publication Standard).

## Definition of Done
Определены жизненный цикл, элементы, привязка к типам происшествий, контрольные точки и правила SOP.
