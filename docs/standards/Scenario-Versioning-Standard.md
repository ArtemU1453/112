# Scenario Versioning Standard — ЕАСУР

- **Status:** APPROVED · Stage 12 · Согласован с Scenario Modeling Standard, Simulation Governance
  Standard, Git Strategy, Exercise Data Governance Standard.

## Назначение
Правила версионирования учебных и моделируемых сценариев (12.2, 12.4). Обеспечивает воспроизводимость,
сравнимость и неизменяемость истории.

## Схема версий
- Идентификатор сценария `SCN-<домен>-NNN`; семантическая версия `MAJOR.MINOR.PATCH`.
- **MAJOR** — несовместимое изменение структуры/начальных условий; **MINOR** — новые события/вводные
  обратно совместимо; **PATCH** — правки без влияния на результаты.
- Версия фиксирует: модель двойника (версия), параметры, seed, наборы синтетических данных.

## Правила
- История версий **неизменяема** (перезапись запрещена); новая правка — новая версия.
- Результаты эксперимента ссылаются на точную версию сценария (воспроизводимость).
- Комбинированные сценарии фиксируют версии компонентов.
- Экзаменационные сценарии — с зафиксированной версией на период сессии (Training Evaluation Standard).

## Хранение
Сценарии — в VCS/репозитории сценариев; крупные наборы данных — в хранилище с версионированием
(Exercise Data Governance / Simulation Analytics).

## Definition of Done
Определены схема версий, правила неизменяемости истории и привязка результатов к версии сценария.
