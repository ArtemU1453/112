# Semantic Search Governance Standard — ЕАСУР

- **Status:** APPROVED · Stage 13 · Согласован с ADR-028, Knowledge Search Architecture, AI Provider
  Abstraction Standard (ADR-021), AI Explainability Standard, AI Assistance Policy.

## Назначение
Правила управления семантическим поиском по базе знаний (13.6): использование ИИ через абстрактный
адаптер, качество, объяснимость и границы применения.

## Принципы
- **Абстракция ИИ:** семантический поиск использует **абстрактный адаптер** (`CAP-AI-001`, ADR-021);
  конкретная модель не фиксируется; без вендор-лока (TAP); self-hosted (AGSDS).
- **Комбинированность:** семантика дополняет полнотекстовый и атрибутивный поиск, не заменяя их
  (Knowledge Search Architecture).
- **Объяснимость и прослеживаемость:** результат сопровождается источником (документ/версия) и, при
  ранжировании ИИ, признаками релевантности; выдача ссылается на утверждённые единицы знаний.
- **Совещательность:** поиск/ранжирование не принимают решений за пользователя; только предоставляют
  и ранжируют материалы.

## Качество и контроль
- Оценка качества поиска (полнота/точность/релевантность) — метрики (Knowledge Analytics).
- Индексируются только опубликованные, актуальные версии; черновики/устаревшее — исключаются или
  помечаются статусом.
- Модель/адаптер обновляются через управляемый процесс; изменения, затрагивающие архитектуру, — RFC→ARB→ADR.

## Границы
Семантический поиск не выдаёт неутверждённые материалы как рекомендации; приватность и безопасность
данных соблюдаются (Security Standards).

## Definition of Done
Определены абстракция ИИ, комбинированность, объяснимость, качество и границы семантического поиска.
