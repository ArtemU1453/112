# Operational Decision Support Standard — ЕАСУР

- **Status:** APPROVED · Stage 5 · Согласован с AI Assistance Policy, Incident Classification
  Standard.

## Назначение (5.7)
Механизм поддержки принятия решений — **advisory**. Система может:
- предлагать варианты **классификации** происшествия;
- показывать **недостающие данные** (обязательные для типа/приоритета);
- предупреждать о **противоречиях** в данных;
- предлагать **дополнительные вопросы** заявителю.

**Окончательное решение всегда принимает диспетчер** (AI Assistance Policy).

## Правила
- Все подсказки — предложения со статусом «предложение»; не изменяют карточку автоматически.
- Недостающие/противоречивые данные определяются по регламентам (Incident Classification Standard,
  Emergency Data Dictionary — обязательные сущности для типа).
- Пороги уверенности предложений ИИ настраиваются; при низкой уверенности — явная пометка.
- Подсказки и их принятие/отклонение журналируются (Operational Audit).

## Деградация
При недоступности ИИ/данных система остаётся работоспособной в ручном режиме (AGSDS); подсказки
отключаются, ручной ввод сохраняется.

## Definition of Done
Определены предложение классификации/недостающие данные/противоречия/вопросы; решение — за
диспетчером; подсказки журналируются.
