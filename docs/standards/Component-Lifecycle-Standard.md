# Component Lifecycle Standard — ЕАСУР

- **Status:** APPROVED · Stage 11 · Согласован с ADR-025, Service Maturity Model, Deprecation Policy
  Standard, Dependency Governance Standard, API Versioning Standard.

## Назначение
Управление жизненным циклом компонентов, сервисов, API, инфраструктуры, библиотек, документации и
архитектурных решений (11.1). Единые фазы и правила перехода.

## Фазы жизненного цикла
```
Proposed → Incubating → Active → Maintained → Deprecated → Retired
```
- **Proposed/Incubating** — предложение/пилот (Innovation/Sandbox), не в критпути.
- **Active** — промышленная эксплуатация, полный уровень зрелости (Service Maturity Model).
- **Maintained** — поддержка без активного развития; только исправления/безопасность.
- **Deprecated** — объявлен вывод; действует Deprecation Policy (сроки, миграция, коммуникация).
- **Retired** — выведен по утверждённому плану миграции.

## Правила перехода
- Переходы фиксируются (владелец, дата, основание); значимые — через RFC/ADR (ARB).
- Прекращение поддержки — **только** с утверждённым планом миграции (запрет этапа).
- Каждая сущность имеет владельца, статус и запись в соответствующем реестре (сервисы/API/зависимости).

## Definition of Done
Определены фазы, правила переходов и владение для всех классов компонентов и архитектурных решений.
