# Event Schema Governance Standard — ЕАСУР

- **Status:** APPROVED · Stage 9 · Дополняет Event Governance Standard (Stage 4) для межведомственных
  событий (Event First).

## Правила
- Каждое событие имеет описанную **схему** (реестр схем) с версией; неописанные события запрещены.
- Эволюция схем — совместимая (аддитивно); несовместимо — новая версия события; потребители
  толерантны к неизвестным полям.
- Схемы событий — канонические (Canonical Data Model); идентификаторы — Canonical Identifier Standard.
- Проверка совместимости схем при публикации (аналог contract testing для событий).
- Реестр схем — источник истины; изменения версионируются и аудируются.

## Definition of Done
Определены реестр схем, версионирование, совместимость и обязательность описания событий.
