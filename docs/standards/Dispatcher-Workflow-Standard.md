# Dispatcher Workflow Standard — ЕАСУР

- **Status:** APPROVED · Stage 5 · Согласован с Emergency Domain Model, AI Assistance Policy,
  UX. Реализуется frontend-dispatcher.

## Рабочее место диспетчера (5.6)
- **Управление вызовом:** приём/удержание/переадресация/завершение (Call Processing Standard).
- **Просмотр карточки:** данные, статус, история, временная шкала, связанные вызовы/наряды.
- **Подтверждение сущностей:** каждая извлечённая сущность (5.4) подтверждается/редактируется/
  отклоняется (Operational State Machine §4); источник и вероятность видимы.
- **Редактирование данных:** с фиксацией изменений (Incident Lifecycle Standard).
- **Быстрые действия и шаблоны:** ускоренные операции по регламенту (без выхода за регламент).
- **История и журнал решений:** все действия и решения диспетчера журналируются (Operational Audit).

## Инварианты
- Диспетчер — единственный, кто утверждает окончательные значения и решения (AI Assistance Policy).
- Рекомендации (5.7/5.8) отображаются как предложения; направление нарядов — только после
  подтверждения диспетчером.
- Быстрые действия не обходят фиксацию изменений и проверки статусов.

## UX-требования
Ясное отображение источника (STT/AI/оператор) и уверенности; выделение недостающих/противоречивых
данных (Decision Support); подтверждение критичных действий; локализация (ru/be/en).

## Definition of Done
Реализованы управление вызовом/просмотр карточки/подтверждение сущностей/редактирование/быстрые
действия/шаблоны/история/журнал решений; окончательные решения — за диспетчером.
