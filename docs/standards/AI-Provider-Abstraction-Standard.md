# AI Provider Abstraction Standard — ЕАСУР

- **Status:** APPROVED · Stage 4 · Согласован с AI Integration Architecture, TAP.

## Цель
Обеспечить независимость платформы от конкретных моделей/поставщиков ИИ. Все интеграции ИИ —
через адаптеры порта `AiProvider` (`CAP-AI-001`).

## Правила
- **Запрещена** привязка к конкретной модели ИИ в прикладном коде; используется абстракция
  возможностей (`AiCapability`: STT/TTS/LLM/OCR/классификация/NER/анализ документов).
- Запрос/ответ — модель-агностичные (`AiRequest`/`AiResponse`).
- Несколько провайдеров сосуществуют; выбор — по требуемой возможности и политике (качество/
  латентность/суверенность).
- Провайдеры предпочтительно self-hosted/локальные (AGSDS); замена — без изменения потребителей.

## Утверждение технологии
Выбор конкретной модели/провайдера фиксируется ADR с обновлением Technology Mapping (TAP).
До утверждения документы/код ссылаются на возможность и порт, а не на продукт.

## Качество и безопасность
Оценка качества (метрики уверенности/точности), пороги и человеческая верификация — на прикладном
этапе (не Stage 4). Данные для ИИ — с учётом ПДн (минимизация, Security Architecture).

## Definition of Done
Определён порт и правила абстракции; привязка к модели запрещена; процесс утверждения технологии
описан.
