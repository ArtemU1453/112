# Call Processing Standard — ЕАСУР

- **Status:** APPROVED · Stage 5 · Согласован с Emergency Domain Model, Operational State Machine,
  Backup Policy. Синхронизирован с telephony-service.

## Жизненный цикл вызова (5.1)
Состояния — `CallStatus` (Operational State Machine §2): RINGING → ACTIVE → ON_HOLD → COMPLETED →
TRANSCRIBED → ANALYZED; не отвеченный → MISSED. Направление — `CallDirection`.

## Приём и очередь
- Каждому вызову присваивается уникальный идентификатор и время начала; определяется линия.
- Очередь ожидания; обработка **параллельных** вызовов; учёт **повторных** вызовов (связывание с
  существующим происшествием/заявителем).
- Удержание (ON_HOLD), переадресация, завершение — с фиксацией в журнале вызовов и Operational
  Timeline (5.9).

## Запись разговора (5.2)
- Запись создаётся при приёме; хранится в файловом хранилище (File Storage Abstraction, ADR-013).
- **Индексирование и поиск** — Search Platform (CAP-SEARCH-001); **экспорт** — по правам и аудиту.
- **Контроль целостности** — контрольные суммы (AGSDS); **журнал доступа** к записям — Operational
  Audit.
- **Политики хранения и архивирование** — сроки удержания по правовым требованиям (EGS/Backup
  Policy). **Удаление записей в обход утверждённых политик запрещено.**

## Передача на обработку речи
По завершении (COMPLETED) публикуется `call.received` (ADR-008) с ссылкой на запись → Speech
Processing (5.3). Результаты возвращаются событием `call.analyzed`.

## Журналирование
Все операции с вызовом (приём/удержание/переадресация/завершение/доступ к записи) журналируются
(Operational Audit) и попадают в Operational Timeline.

## Definition of Done
Определены приём/очередь/состояния/параллельные и повторные вызовы; запись/хранение/поиск/экспорт/
целостность/доступ/удержание/архивирование; запрет удаления в обход политик.
