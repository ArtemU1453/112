# Backlog Structure — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0

Единый Backlog — единственный источник рабочих заданий (Constitution, ARTICLE 15).

## Иерархия элементов

```
Program
 └─ Epic
     └─ Feature
         └─ Story
             └─ Task
```
Плюс сквозные типы: **Bug**, **Spike**, **Technical Debt**.

| Тип | Назначение | Горизонт |
|-----|------------|----------|
| Program | Стратегическая инициатива (например, «Ядро платформы») | Кварталы |
| Epic | Крупная функциональная область (например, «Диспетчеризация») | Недели–месяцы |
| Feature | Законченная функция, ценность для пользователя | Дни–недели |
| Story | Пользовательский сценарий (as a … I want … so that …) | Дни |
| Task | Технический шаг реализации Story | Часы–день |
| Bug | Дефект относительно ожидаемого поведения | По приоритету |
| Spike | Исследование/прототип для снятия неопределённости | Ограничен по времени |
| Technical Debt | Зарегистрированный долг для планового устранения | По приоритету |

## Обязательные поля элемента

Уникальный идентификатор (`INC-001`, `DISP-014`, …), название, описание, бизнес-ценность,
техническое описание, DoR, DoD, зависимости, приоритет, оценка сложности, ссылки на
Epic/Feature. Для Bug — шаги воспроизведения, ожидаемое/фактическое; для Spike — вопрос и
критерий завершения; для Technical Debt — причина, влияние, предлагаемое устранение.

## Идентификаторы (префиксы по контексту)

`INC` (incident), `DISP` (dispatch), `TEL` (telephony), `GIS` (gis), `AUTH` (auth),
`AUD` (audit), `NOTE` (notification), `RT` (realtime), `AI` (ai), `GW` (gateway),
`FE` (frontend), `INF` (infrastructure/DevOps), `DOC` (документация).

## Трассируемость

Требование → Feature → Story → Task → Код → Тесты → Документация (Requirements Management
Standard). Каждая ветка и PR ссылаются на идентификатор задачи (Git Strategy).

## Правила

- Работа ведётся только через элементы Backlog; за один инкремент — одна Task (ARTICLE 16).
- Технический долг и риски не скрываются — заводятся как Technical Debt / связанные задачи
  (ARTICLE 19, Risk Register).
