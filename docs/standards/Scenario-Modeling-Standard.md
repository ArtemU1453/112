# Scenario Modeling Standard — ЕАСУР

- **Status:** APPROVED · Stage 12 · Согласован с ADR-026, Simulation Architecture, Scenario Versioning
  Standard, Simulation Data Model, Emergency domain (Stages 5–6).

## Назначение
Правила описания и моделирования сценариев ЧС в среде цифрового двойника (12.2). Обеспечивает
единообразие, воспроизводимость и совместимость сценариев с моделью данных симуляции.

## Типы сценариев
Пожары · ДТП · аварии · химические происшествия · природные ЧС · техногенные аварии · массовые
происшествия · **комбинированные события** (композиция нескольких типов). Классификация согласована с
доменной моделью инцидентов (IncidentType/Priority, Stage 5).

## Структура сценария (обязательные элементы)
- **Идентификация и версия** (Scenario Versioning Standard): id, версия, автор, дата, статус.
- **Начальные условия:** территория/объекты/инфраструктура двойника, погода/время, доступные ресурсы.
- **События и триггеры:** временная шкала событий, эскалации, вводные (injects) для учений.
- **Ресурсная конфигурация:** подразделения/техника/зоны ответственности (из двойника, 12.1).
- **Целевые метрики:** критерии эффективности/оценки (Simulation Analytics, Training Evaluation).
- **Данные:** только синтетические/обезличенные (Synthetic Data Standard); реальные — запрещены.

## Правила
- Сценарий детерминирован при фиксированном seed (воспроизводимость, Simulation Validation Standard).
- Сценарий версионируется; изменения — новая версия (без перезаписи истории).
- Комбинированные сценарии собираются из версионированных компонентов.
- Учебные и реальные инциденты не смешиваются (маркировка среды).

## Definition of Done
Определены типы, структура и правила моделирования сценариев с воспроизводимостью и версионированием.
