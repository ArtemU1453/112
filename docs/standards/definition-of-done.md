# Definition of Done (DoD) — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0

Общий DoD проекта. Задача считается завершённой (`→ Done`) только при выполнении всех условий.

## Критерии завершения

- [ ] Реализованы все критерии приёмки задачи.
- [ ] Код соответствует Coding Standards и Development Guidelines.
- [ ] Архитектура не нарушена: соответствие Vision, Constitution, Architecture Principles,
      ADR, RFC, правилам зависимостей и границам контекстов (Architecture Compliance).
- [ ] Написаны и проходят тесты требуемых уровней (Unit обязательно; Integration/Contract/E2E —
      по DoR). Функциональность без тестов не считается завершённой (ARTICLE 8).
- [ ] Пройден внутренний Code Review (ARTICLE 17).
- [ ] Пройдены Quality Gates: сборка, статический анализ, security scan, покрытие не ниже порога.
- [ ] API соответствует OpenAPI; схема БД соответствует документации и миграциям.
- [ ] Обновлена документация (Vision/Architecture/ADR/RFC/API/Database/Security/Deployment/
      Testing — в затронутой части); документация не отстаёт от кода (ARTICLE 9, 18).
- [ ] Обеспечена наблюдаемость: логирование, метрики, health checks (ARTICLE 11).
- [ ] Соблюдена безопасность: минимальные привилегии, нет секретов/ПДн в коде и логах (ARTICLE 10).
- [ ] Обратная совместимость соблюдена или изменение проведено через RFC (ARTICLE 6).
- [ ] Выявленный технический долг/риски зафиксированы отдельными задачами Backlog (ARTICLE 19).
- [ ] Зафиксирован журнал решений (изменённые компоненты, затронутые ADR/RFC, решения).

Если хотя бы одно условие не выполнено, задача считается незавершённой.
