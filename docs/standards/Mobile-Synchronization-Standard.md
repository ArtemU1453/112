# Mobile Synchronization Standard — ЕАСУР

- **Status:** APPROVED · Stage 7 · ADR-017 · Дополняет Offline Synchronization Standard (клиентская часть).

## Клиентские правила синхронизации
- Триггеры: восстановление связи, периодический таймер, ручной запуск, приоритетные события.
- Приоритизация: критичные данные (статусы/приказы) — первыми; медиа — фоново.
- Батчинг и дельта; ограничение размера пакета под слабый канал; докачка при обрыве.
- Идемпотентные запросы (ключи) — без дублей; экспоненциальные повторы.
- Индикация состояния синхронизации пользователю; журнал синхронизации.

## Совместимость
Единый контракт синхронизации (`mobile-sync-api.yaml`); версии контракта — API Lifecycle Management
Standard.

## Definition of Done
Определены триггеры/приоритеты/батчинг/идемпотентность/повторы/индикация клиентской синхронизации.
