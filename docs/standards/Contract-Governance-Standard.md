# Contract Governance Standard — ЕАСУР

- **Status:** APPROVED · Stage 9 · Согласован с API Lifecycle Management / API Versioning Standards,
  Integration Governance (Stage 4).

## Управление контрактами
Каждый интеграционный интерфейс имеет контракт (OpenAPI/gRPC proto) — источник истины (Contract
First). Контракт версионируется, имеет владельца, статус жизненного цикла и записи в каталоге API.

## Правила (обязательные)
- Изменение контракта — только с новой версией **или** документированным планом совместимости
  (запрет изменения API без версии/плана).
- Несовместимые изменения — новая мажорная версия + Contract Testing + сертификация.
- Обратная совместимость по умолчанию; депрекация — с уведомлением и сроком (SLA).
- Неописанные форматы сообщений запрещены; все обмены — по контракту.
- Изменения контрактов проходят через RFC/ADR при архитектурном влиянии.

## Процесс изменения
Design → Review (совместимость) → Approve → Publish (версия) → Test → Operate → Deprecate → Retire.

## Definition of Done
Определены владение/версионирование/совместимость/жизненный цикл контрактов и запреты.
