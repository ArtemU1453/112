# Rule Authoring Standard — ЕАСУР

- **Status:** APPROVED · Stage 13 · Согласован с ADR-028, Rules Catalog Specification, Regulatory
  Document Standard, AI Explainability Standard, Decision Intelligence Standard (Stage 5).

## Назначение
Правила создания правил каталога (13.4): структура, объяснимость, обязательная привязка к нормативному
основанию, условия применения, исключения, приоритеты.

## Структура правила (обязательные элементы)
| Элемент | Описание |
|---------|----------|
| ID | `RULE-<домен>-NNN`, версия |
| Тип | классификационное / рекомендательное |
| Условие применения | когда правило срабатывает (контекст/атрибуты) |
| Действие/рекомендация | что предлагается (совещательно) |
| Исключения | когда правило не применяется |
| Приоритет | порядок при конфликте правил |
| **Основание** | ссылка на нормативный документ/SOP (версия) — **обязательно** |
| Объяснение | человекочитаемое обоснование |

## Инварианты
- **Каждое правило имеет документированное основание** (нормативную ссылку); применение правил без
  основания запрещено (запрет этапа).
- Правила **объяснимы**: формируют человекочитаемое объяснение причины рекомендации.
- Правила совещательны: не изменяют рабочие процессы автоматически; окончательное решение — за человеком.
- Конфликты разрешаются по приоритетам; исключения задокументированы.

## Версионирование и трассируемость
Правила версионируются (Document Versioning); связь «правило → основание → версия норматива»
неизменяема; история изменений не удаляется.

## Definition of Done
Определены структура, обязательное основание, объяснимость, приоритеты и версионирование правил.
