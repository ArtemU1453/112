# Upgrade Compatibility Standard — ЕАСУР

- **Status:** APPROVED · Stage 11 · Согласован с ADR-025, API Versioning Standard, Contract Governance
  Standard, Modernization Standard, Dependency Governance Standard.

## Назначение
Правила анализа и обеспечения совместимости при обновлениях (11.4, 11.5): компоненты, API, данные,
зависимости, среда выполнения. Обновление/внедрение без анализа совместимости запрещено.

## Виды совместимости
- **API-совместимость:** обратная совместимость контрактов (API Versioning); несовместимое — новая версия.
- **Совместимость данных:** миграции схем обратимы и идемпотентны; сохранность и целостность данных.
- **Совместимость зависимостей:** транзитивные конфликты, ABI/API библиотек, среда выполнения.
- **Протокольная/интеграционная:** каноническая модель и версии контрактов межведомственных обменов.

## Анализ совместимости (обязательный перед обновлением)
Матрица «было → станет»; влияние на клиентов/интеграции/данные; план миграции и отката; проверка на
staging (окружение идентично prod, EGS); прогон контрактных тестов.

## Правила
- Несовместимое изменение — только через версионирование + план миграции + депрекация старого (Deprecation Policy).
- Критпуть не переводится на несовместимую версию без подтверждённой совместимости и отката.

## Definition of Done
Определены виды совместимости, обязательный анализ и правила для обновлений и миграций.
