# Internal SDK Governance Standard — ЕАСУР

- **Status:** APPROVED · Stage 4 · Согласован с Engineering Handbook, Coding Standards, EAGF.

## Цель
Определить правила разработки, версионирования и обязательного использования внутреннего SDK
(`platform-sdk`) и общих библиотек всеми сервисами.

## Обязательность
Каждый новый сервис **обязан** подключать SDK и использовать единые возможности: безопасность,
журналирование, конфигурация, аудит, ошибки, мониторинг, трассировка (`PlatformModule`).
Дублирование инфраструктурного кода запрещено (Constitution ARTICLE 7; Service Maturity Model).

## Версионирование и совместимость
- SemVer для `platform-*`; версии — BOM (`libraries/build/bom`) и каталог версий (TVMS).
- Обратная совместимость по умолчанию; несовместимые изменения — мажорная версия + миграция +
  RFC/ADR при влиянии на архитектуру.
- Депрекация — с переходным периодом и записью в CHANGELOG.

## Качество
Общие библиотеки покрыты тестами; проходят Quality Gates (Coding Standards, Testing Strategy).
Публичный API SDK документирован (SDK Documentation, Common Libraries Guide).

## Управление изменениями
Изменения SDK — через PR с ревью владельцев; влияющие на все сервисы — с анализом влияния.

## Definition of Done
SDK и библиотеки определены, версионируются централизованно, обязательны к использованию,
документированы и покрыты тестами.
