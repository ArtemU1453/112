# Quality Gates — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0

Обязательные проверки перед завершением задачи и слиянием (Constitution, ARTICLE 5).
Реализуются стадиями CI/CD (ADR-006) и процедурой Code Review.

## Обязательные ворота

| # | Ворота | Что проверяется | Где |
|---|--------|-----------------|-----|
| QG1 | Code Review | Архитектура, стандарты, безопасность, производительность, тестируемость | PR |
| QG2 | Unit Tests | Модульные тесты проходят; покрытие не ниже порога | CI `test` |
| QG3 | Integration Tests | Testcontainers (PostgreSQL/Kafka), контрактные проверки | CI `test` |
| QG4 | Static Analysis | Линтеры/анализаторы без новых критических нарушений | CI `test` |
| QG5 | Security Scan | Уязвимости зависимостей/образов, секреты в коде | CI |
| QG6 | Documentation Review | Синхронизация Vision/Architecture/ADR/API/DB/Security/Testing | PR |
| QG7 | Architecture Review | Для архитектурно значимых изменений — соответствие Baseline/RFC | PR |

## Пороговые значения (Baseline v1.0)

- Покрытие доменного кода (domain/application) — не ниже согласованного порога проекта;
  снижение покрытия ниже порога блокирует слияние.
- Ноль новых критических/высоких предупреждений статического анализа и security scan
  без явного обоснования и зарегистрированного техдолга.
- Ноль секретов в коде (проверка секрет-сканером).

## Правила прохождения

- Все применимые ворота должны быть зелёными; пропуск ворот запрещён (нельзя жертвовать
  проверками ради скорости).
- Невозможность пройти ворота из-за внешних причин фиксируется как риск/техдолг и согласуется
  до слияния.
- Для production-развёртывания — ручной gate (ADR-006) поверх пройденных QG.

## Связь с DoD

Прохождение Quality Gates — обязательная часть Definition of Done. Невыполнение любого
применимого ворота означает, что задача не завершена.
