# Shared Domain Kernel Standard — ЕАСУР

- **Status:** APPROVED · Stage 4 · Согласован с Constitution, Architecture Baseline, DMS, EAGF.

## Цель
Определить, какой код может быть **общим** между сервисами (shared kernel), а какой обязан
оставаться в границах bounded context. Предотвратить размывание границ и связанность через общий
доменный код.

## Область применения
Все общие библиотеки (`libraries/platform-*`) и любые переиспользуемые артефакты.

## Правила
- **Разрешено в shared kernel:** не-доменный инфраструктурный код (ошибки, DTO/пагинация,
  валидация общего назначения, время, локализация, логирование, безопасность-контекст,
  идентификаторы, порты-абстракции), контрактные схемы событий/канонической модели.
- **Запрещено в shared kernel:** доменные агрегаты/инварианты конкретного контекста, доменные
  сервисы, бизнес-правила, общий слой доступа к данным между сервисами (Constitution ARTICLE 3, 7;
  System Structure).
- Общий код изменяется с сохранением обратной совместимости; несовместимо — мажорная версия + план
  миграции (Dependency Governance Standard).
- Автономность bounded contexts приоритетнее DRY между ними (дублирование контрактных DTO допустимо).

## Управление
Изменения shared kernel — через PR с ревью владельцев (`CODEOWNERS`); архитектурно значимые —
RFC/ADR. Нарушение границ — архитектурный дрейф (Architecture Baseline).

## Definition of Done
Определены границы общего кода; общие библиотеки не содержат доменной/бизнес-логики; правила
версионирования и совместимости зафиксированы.
