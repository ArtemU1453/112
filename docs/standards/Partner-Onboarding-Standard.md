# Partner Onboarding Standard — ЕАСУР

- **Status:** APPROVED · Stage 9 · Согласован с Partner Integration Guide, Integration Certification
  Standard.

## Процесс подключения партнёра
1. Регистрация внешней системы в реестре (обязательна; без регистрации подключение запрещено).
2. Согласование контрактов/версий (Contract First) и канонической модели.
3. Выпуск сертификатов, настройка mTLS/OAuth2/scope (ADR-023).
4. Тестирование в песочнице (Partner Sandbox): тестовые данные, эмуляция, contract testing.
5. Сертификация интеграции; ввод в продуктив через платформу.

## Правила
Минимальные привилегии; обязательства по совместимости/уведомлению об изменениях; отзыв доступа —
через реестр (governance). Все шаги журналируются.

## Definition of Done
Определён сквозной процесс онбординга: регистрация → контракты → безопасность → песочница →
сертификация → продуктив.
