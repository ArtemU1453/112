# Canonical Data Model Standard — ЕАСУР

- **Status:** APPROVED · Stage 4 · Согласован с DMS, Integration/Messaging Architecture.

## Цель
Определить каноническую модель данных для обмена между подсистемами и внешними системами —
единые, версионируемые представления общих понятий, независимые от внутренних моделей контекстов.

## Принципы
- Внутренние доменные модели bounded context **не** экспонируются напрямую; обмен — через
  каноническую модель (снижает связанность).
- Преобразование внутренняя↔каноническая модель выполняется на границах (адаптеры/маперы,
  `platform-commons.mapping`).
- Каноническая модель версионируется (SemVer); эволюция — аддитивно; несовместимо — новая версия.

## Состав
- Канонические представления общих сущностей (например, пользователь, подразделение, адрес,
  геокоордината) — как контрактные схемы (`contracts/shared`), не как общий доменный код (Shared
  Domain Kernel Standard).
- Идентификаторы и справочники — единые (Localization/справочники User Management).

## Управление
Изменения канонической модели — через RFC/ADR при влиянии на интеграции; синхронизация контрактов
и Technology Mapping. Обратная совместимость по умолчанию.

## Definition of Done
Определены принципы, версионирование и границы преобразования; каноническая модель отделена от
внутренних моделей контекстов.
