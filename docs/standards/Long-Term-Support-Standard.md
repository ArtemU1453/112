# Long-Term Support Standard — ЕАСУР

- **Status:** APPROVED · Stage 10 · Согласован с ADR-024, Toolchain Governance Standard, Dependency
  Governance Standard, Release Governance Standard, Platform Operations Framework.

## Назначение
Правила долгосрочного сопровождения национальной системы после ввода в эксплуатацию (10.8, 10.10):
поддержка версий, обновления, безопасность, эволюция без потери непрерывности и суверенности.

## Принципы
- **Управляемая эволюция:** изменения — только через RFC/ADR и релизный процесс; архитектура остаётся
  SSOT (Architecture Baseline, Freeze/Drift-контроль).
- **Версии и жизненный цикл:** сопровождение версий по каталогу (TVMS); плановое обновление
  языков/платформ/зависимостей (LTS-линии), контроль EndOfLife.
- **Безопасность непрерывно:** мониторинг уязвимостей, патчи, ротация секретов/сертификатов (PKI),
  повторные сертификации при значимых изменениях.
- **Совместимость:** обратная совместимость контрактов и данных; депрекация — по политике версий.
- **Суверенность:** self-hosted/air-gapped сохраняется (AGSDS); отсутствие вендор-лока (TAP).
- **Знания:** актуализация документации и базы знаний; преемственность команд.

## Обслуживаемость
Регулярные окна обслуживания (Maintenance Window Standard), учения непрерывности (Business Continuity
Validation), периодический аудит соответствия архитектуре (EAGF).

## Definition of Done
Определены принципы долгосрочного сопровождения, обновлений, безопасности и совместимости.
