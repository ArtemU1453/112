# Data Lineage Standard — ЕАСУР

- **Status:** APPROVED · Stage 8 · Согласован с Data Quality / Metrics Catalog Standards.

## Прослеживаемость происхождения (8.6/8.9)
Для каждого аналитического показателя/поля витрины прослеживается цепочка: источник (событие/
таблица) → трансформации (ETL/ELT) → витрина → метрика/отчёт/дашборд.

## Правила (обязательные)
- **Происхождение каждого показателя прослеживается** (обязательная проверка этапа); скрывать
  происхождение запрещено.
- Трансформации документированы (методика/код версионируются).
- Lineage доступен для аудита и для объяснимости выводов ИИ (перечень использованных данных).
- Изменения источников/трансформаций отражаются в lineage.

## Definition of Done
Определена сквозная прослеживаемость от источника до показателя; документированность трансформаций.
