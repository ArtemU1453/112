# RFC-0010 — Платформа знаний и поддержки решений

- **Статус:** Closed (Approved → Implemented → Verified)
- **Автор:** Knowledge Governance / Architecture Review Board (Stage 13)
- **Дата:** 2026-07-21
- **Связанные ADR:** ADR-028 (платформа знаний и поддержки решений)
- **Связанные документы:** Knowledge Platform Architecture, стандарты и руководства Stage 13

## 1. Причина изменения
Требуется централизованная платформа управления знаниями, нормативными документами, справочниками и
правилами поддержки решений: единый источник, версионирование, контролируемое распространение,
интеллектуальный поиск и объяснимая поддержка решений — без изменения бизнес-логики ранее
реализованных модулей.

## 2. Существующее решение
Stages 1–12 создали систему, эволюцию и изолированную платформу моделирования. Знания (нормативы/SOP/
справочники/правила) не были собраны в единый управляемый реестр; `CAP-SEARCH-001` оставалась Proposed.

## 3. Предлагаемое решение
Единая платформа знаний (ADR-028): реестр документов, управление нормативными документами и SOP,
каталог правил (с основаниями), справочная информация, интеллектуальный поиск (полнотекст/атрибуты/
семантика через адаптер ИИ), слой поддержки решений (объяснимый, совещательный), распространение
изменений, компетенции/аттестация и аналитика знаний.

## 4. Impact Analysis
- Модули/сервисы прод: **бизнес-логика не меняется** — платформа предоставляет справочный слой знаний
  (только чтение утверждённых версий); изменения процессов — через RFC→ARB→ADR.
- API/данные: реестр знаний/SOP/правил/справочников — собственная модель; оперативные схемы не затронуты.
- Поиск: `CAP-SEARCH-001` → Approved (self-hosted, абстракция); семантика — `CAP-AI-001` (адаптер).
- Безопасность: RBAC, аудит доступа/ознакомления, неизменяемая история; Zero Trust.

## 5. Compatibility Check
Обратная совместимость полная; контракты/схемы оперативных сервисов не изменяются; интеграция —
аддитивная (справочный слой). Документация синхронизирована; история изменений не удаляется.

## 6. Риски
Актуальность и качество знаний; риск применения неутверждённого. Снижение — Knowledge Quality/Retention,
Regulatory Document/Document Versioning стандарты, обязательное ознакомление, объяснимость правил.

## 7. Migration Plan
Учреждение реестра знаний → загрузка утверждённых нормативов/SOP/справочников → каталог правил с
основаниями → поиск (полнотекст/атрибуты/семантика) → слой поддержки решений → распространение и
аналитика → компетенции/аттестация.

## 8. Rollback Plan
Платформа аддитивна и может быть отключена без влияния на оперативный контур; версии документов/правил —
в реестре с неизменяемой историей; откат публикаций — через статусы (не удаление).
