# Architecture Review Handbook — ЕАСУР

- **Status:** APPROVED · Stage 11 · Реализует ADR-025, Architecture Evolution Standard, Continuous
  Compliance Standard (подэтап 11.2). Руководство работы Architecture Review Board (ARB).

## 1. Назначение
Как организован непрерывный архитектурный аудит и рассмотрение изменений: состав ARB, ритм ревью,
критерии, работа с дрейфом и соответствием.

## 2. Architecture Review Board (ARB)
Постоянный орган архитектурной ответственности. Функции: рассмотрение RFC, периодический аудит,
утверждение модернизации/депрекации/внедрения технологий, контроль принципов Enterprise Architecture.

## 3. Виды ревью
| Вид | Триггер | Фокус |
|-----|---------|-------|
| Change Review (RFC) | Поступление RFC | Impact/Compatibility/Migration/Rollback, соответствие Baseline |
| Periodic Audit | Расписание (ежеквартально) | Дрейф, соблюдение ADR/RFC, техдолг, соответствие принципам EA |
| Release Architecture Review | Крупный релиз | Архитектурные последствия релиза |
| Incident-driven Review | SEV-1/2, повторяющиеся инциденты | Архитектурные корневые причины |

## 4. Проверяемые аспекты (архитектурный аудит)
- **Дрейф:** расхождение реализации и утверждённой архитектуры (валидатор архитектуры + ручной обзор).
- **Соблюдение ADR/RFC:** решения применяются как утверждено; отклонения — регистрируются.
- **Принципы EA:** микросервисы/гексагон/CQRS, SSOT, TAP, Zero Trust, суверенность, обратная совместимость.
- **Техдолг:** тренд и критические записи (Technical Debt Register).
- **Соответствие стандартам:** engineering/domain/ops стандарты.

## 5. Порядок Change Review
RFC → проверка полноты (шаблон) → анализ влияния/совместимости → решение (Approve/Approve с условиями/
Reject) → ADR → синхронизация документации → верификация после реализации.

## 6. Результаты и фиксация
Протокол ревью, решения, замечания, действия и сроки — неизменяемо (аудит, EAGF). Выявленные отклонения
→ Technical Debt Register / план устранения. Итоги периодического аудита — во входные данные стратегии.

## 7. Связи
ADR-025 · Architecture Evolution / Continuous Compliance Standards · RFC-процесс · EAGF · Technical
Debt Register · Continuous Improvement Framework.
