# Knowledge Management Guide — ЕАСУР

- **Status:** APPROVED · Stage 11 · Реализует Knowledge Governance Standard (подэтап 11.6).
  Практическое руководство по базе знаний системы.

## 1. Назначение
Организация архитектурных и эксплуатационных знаний: структура, захват, публикация, пересмотр,
преемственность команд. Архитектурная документация не удаляется (запрет этапа).

## 2. Структура базы знаний
| Раздел | Источник (SSOT) |
|--------|-----------------|
| Архитектура | Vision, Architecture.md, принципы, ADR/RFC, стандарты |
| Возможности/технологии | Technology Mapping, каталог версий |
| Эксплуатация | Operations/Support Handbook, Production Runbook, платформенные runbooks |
| Инструкции/руководства | docs/*/*-guide, developer-experience |
| FAQ | Разделы FAQ в руководствах; частые обращения (Support) |
| Lessons Learned | Разборы SEV-1/2 и крупных релизов |
| Best Practices | Инженерные/эксплуатационные практики, engineering-handbook |

## 3. Процессы
- **Захват:** по итогам инцидентов, ревью ARB, релизов, модернизаций.
- **Классификация и публикация:** в соответствующий раздел docs/ с владельцем и статусом.
- **Пересмотр:** периодический (актуальность ≥ реализации); устаревшее — статус, не удаление.
- **Преемственность:** онбординг, обучение (Training Handbook), передача владения.

## 4. Lessons Learned (обязательно)
После SEV-1/2 и крупных релизов: контекст, что произошло, корневая причина, что сработало/нет, действия
и изменения процессов/стандартов. Ссылки на инциденты/RFC/ADR.

## 5. Правила
SSOT и версионирование; неудаление архитектурных знаний; актуализация как часть Definition of Done;
единый стиль (Documentation Standards/шаблоны).

## 6. Связи
Knowledge Governance Standard · Documentation Standards · engineering-handbook · Operations/Support
Handbook · Continuous Improvement Framework.
