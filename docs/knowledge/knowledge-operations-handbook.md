# Knowledge Operations Handbook — ЕАСУР

- **Status:** APPROVED · Stage 13 · Эксплуатация платформы знаний. Согласован с ADR-028, Platform
  Operations Framework (Stage 3), Knowledge Publication/Retention Standards, Operations Handbook (Stage 10).

## 1. Назначение
Руководство по эксплуатации национальной платформы знаний: ведение реестра, публикация и распространение,
поиск, поддержка решений, аналитика, безопасность, обслуживание. Бизнес-логика оперативных модулей не
затрагивается.

## 2. Регламентные операции
| Операция | Действия |
|----------|----------|
| Ведение реестра | Создание/редактирование единиц знаний, статусы, версии (Lifecycle/Versioning) |
| Согласование/публикация | Утверждение и публикация (Knowledge Publication Standard) |
| Распространение | Уведомления, обязательное ознакомление, контроль журнала (13.8) |
| Поиск/индексация | Поддержка индексов; индексируется только утверждённое (13.6) |
| Поддержка решений | Проверка рекомендаций/объяснений из SOP/правил (13.7) |
| Аналитика/актуализация | Обзор метрик, устаревающего, пробелов (13.10) |
| Компетенции/аттестация | Программы/тесты, сроки переаттестации (13.9) |

## 3. Наблюдаемость
Метрики использования/поиска/ознакомления; логи и аудит доступа/публикаций; дашборды (по образцу
Stage 3/8). Контроль просрочек ознакомления и переаттестации.

## 4. Безопасность и целостность
RBAC (ADR-007); аудит доступа/ознакомления; секреты в Vault; неизменяемая история (Knowledge Retention);
резервное копирование (Backup Policy). Только чтение утверждённого в оперативном применении.

## 5. Инварианты (обязательные проверки этапа)
Все нормативные документы версионируются; правила связаны с основанием; изменения распространяются
контролируемо; поиск поддерживает полнотекст и семантику; рекомендации сопровождаются объяснениями;
полная трассируемость изменений.

## 6. Обслуживание
Пересмотр по срокам; актуализация по рекомендациям аналитики; изменения — через утверждённый процесс;
архитектурные/процессные — через RFC→ARB→ADR. История не удаляется.
