# Rules Catalog Specification — ЕАСУР

- **Status:** APPROVED · Stage 13 · Реализует Rule Authoring Standard (13.4). Согласован с ADR-028,
  Decision Intelligence Standard (Stage 5), AI Explainability Standard, Regulatory Document Standard.

## 1. Назначение
Спецификация каталога правил: классификационные и рекомендательные правила с условиями применения,
исключениями, приоритетами и **обязательной ссылкой на нормативное основание**.

## 2. Модель правила
| Поле | Описание | Обязательность |
|------|----------|----------------|
| `id` | `RULE-<домен>-NNN` | да |
| `version` | Версия правила | да |
| `type` | classification / recommendation | да |
| `condition` | Условие применения (контекст/атрибуты) | да |
| `action` | Рекомендация/классификация (совещательно) | да |
| `exceptions` | Условия неприменения | нет |
| `priority` | Приоритет при конфликте | да |
| `basis` | Ссылка на нормативный документ/SOP (версия) | **да** |
| `explanation` | Человекочитаемое обоснование | **да** |

## 3. Инварианты (обязательные проверки этапа)
- **Каждое правило связано с нормативным основанием** (`basis`); правила без основания недопустимы.
- Правила **объяснимы**: выдают человекочитаемое объяснение причины рекомендации.
- Правила **совещательны**: не изменяют рабочие процессы автоматически; решение — за человеком.

## 4. Разрешение конфликтов
При срабатывании нескольких правил применяется приоритет; исключения снимают применимость; результат и
цепочка правил — прослеживаемы (для объяснения пользователю).

## 5. Версионирование и связи
Правила версионируются (Document Versioning Standard); связь «правило → основание → версия норматива/SOP»
неизменяема; удаление истории запрещено. Правила используются слоем поддержки решений (13.7) и поиском.
