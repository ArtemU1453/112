# Knowledge Search Architecture — ЕАСУР

- **Status:** APPROVED · Stage 13 · Реализует Semantic Search Governance Standard (13.6). Согласован с
  ADR-028, AI Provider Abstraction Standard (ADR-021), Knowledge Platform Architecture.

## 1. Назначение
Архитектура интеллектуального поиска по базе знаний: полнотекстовый, атрибутивный и семантический поиск,
фильтрация, ранжирование, поиск по связанным документам.

## 2. Компоненты
| Компонент | Функция | Абстракция |
|-----------|---------|------------|
| Index Manager | Индексация опубликованных версий единиц знаний | `CAP-SEARCH-001` (self-hosted) |
| Full-text Search | Полнотекстовый поиск | `CAP-SEARCH-001` |
| Attribute Search | Поиск по атрибутам/метаданным/статусам | `CAP-SEARCH-001` |
| Semantic Search | Семантический поиск через **абстрактный адаптер ИИ** | `CAP-AI-001` (ADR-021) |
| Ranking | Ранжирование результатов (в т.ч. признаки релевантности) | Semantic Search Governance |
| Related Docs | Поиск по связям (основания/ссылки/зависимости) | Knowledge model |

## 3. Принципы
- **Комбинированность:** семантика дополняет полнотекст/атрибуты, не заменяя их.
- **Абстракция:** поисковый движок и модель семантики — self-hosted, не фиксируются (TAP); без вендор-лока.
- **Индексируется только утверждённое:** черновики/устаревшее исключаются или помечаются статусом.
- **Объяснимость:** результаты ссылаются на источник (документ/версия); при ИИ-ранжировании — признаки
  релевантности; поиск **совещателен** (не принимает решений).

## 4. Поток запроса
Запрос → разбор (полнотекст/атрибуты/семантика) → выборка из индекса утверждённых версий → фильтрация →
ранжирование → результаты со ссылками на источники и связанные документы.

## 5. Качество и безопасность
Метрики полноты/точности/релевантности (Knowledge Analytics); RBAC и приватность (Security Standards);
доступ по ролям; аудит запросов при необходимости.

## 6. Связи
Semantic Search Governance Standard · Knowledge Platform Architecture · AI Provider Abstraction (ADR-021) ·
Decision Support Layer · Technology Mapping (`CAP-SEARCH-001` Approved, `CAP-AI-001`).
