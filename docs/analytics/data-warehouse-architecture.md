# Data Warehouse Architecture — ЕАСУР

- **Status:** APPROVED · Stage 8 · ADR-019 · Подчиняется Analytics Architecture, DGS.

## Назначение
Аналитическое хранилище (DWH, `CAP-DWH-001`) — исторические данные, агрегированные показатели,
витрины данных, каталог данных, управление качеством. Отделено от оперативного контура.

## Слои DWH
```
Источники (Kafka-события + снапшоты оперативных БД)
  → Staging (сырые данные, неизменяемые)
  → Core (интегрированная историческая модель, версии/срезы)
  → Data Marts (витрины: вызовы/инциденты/подразделения/KPI)
  → Каталог данных + метрики качества + lineage
```

## Наполнение (ETL/ELT)
- Потоковая загрузка из событий Kafka (ADR-008); периодические снапшоты БД.
- Трансформации/витрины — декларативно (например, dbt), с документированной методикой.
- Идемпотентность загрузки; восстановимость (перечитывание событий).

## Историчность и неизменяемость
Исторические данные **неизменяемы** (append/slowly changing dimensions); изменения — только с
**полным аудитом** (запрет этапа). Срезы фиксируются для воспроизводимости отчётов.

## Каталог данных и качество
Каталог витрин/полей/владельцев (Metrics Catalog Standard); прослеживаемость происхождения
(Data Lineage Standard); контроль качества (Data Quality Standard); мастер-данные (Master Data
Governance Standard).

## Retention
Сроки хранения аналитических данных — Data Retention for Analytics Standard; резервирование —
Backup Policy; суверенно/self-hosted (AGSDS).
