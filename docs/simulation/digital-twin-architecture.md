# Digital Twin Architecture — ЕАСУР

- **Status:** APPROVED · Stage 12 · Подчиняется Architecture.md, ADR-026, Digital Twin Governance
  Standard, Simulation Security Standard. Изолированный контур, вне промышленной эксплуатации.

## 1. Назначение
Цифровая модель («двойник») национальной системы реагирования: подразделения, техника, объекты,
территории, зоны ответственности, инфраструктура, логистика (12.1). Основа для моделирования, обучения
и стратегического планирования.

## 2. Принципы
API First · Contract First · **изоляция от прод** (ADR-026) · однонаправленный офлайн-импорт справочной
модели · синтетическая оперативная динамика (Synthetic Data Standard) · версионирование состояний ·
Technology Abstraction (self-hosted, `CAP-DTWIN-001`) · суверенность (AGSDS).

## 3. Состав модели двойника
| Домен | Содержание | Источник |
|-------|-----------|----------|
| Подразделения | Пожарные/аварийные/медицинские части, штаты | Офлайн-снимок реестра ресурсов (Stage 6) |
| Техника | Типы/количество/готовность | Офлайн-снимок + синтетическая динамика |
| Объекты | Критическая инфраструктура, объекты риска | Справочная модель |
| Территории | Административные/оперативные границы | GIS (Stage 6, обезличенные слои) |
| Зоны ответственности | Привязка подразделений к территориям | Reference model |
| Инфраструктура | Дороги/сеть/связь (модельно) | Геоданные/OSM-совместимые |
| Логистика | Маршруты/время прибытия | Routing (CAP-ROUTING-001, self-hosted) |

## 4. Слои архитектуры
- **Reference Model Import** — приём офлайн-снимка справочной модели (без онлайн-связи с прод).
- **Twin State Store** — версионированное состояние двойника (Simulation Data Model).
- **Twin Domain Model** — сущности и связи (согласованы с доменом Stage 5–6, но изолированы).
- **Twin API** — доступ движка сценариев/симуляции/аналитики к модели (внутри контура).

## 5. Актуализация и версии
Обновление структуры — из офлайн-снимка (Digital Twin Governance); каждое состояние версионируется;
прошлые версии архивируются (не удаляются). Обратная запись в прод отсутствует.

## 6. Изоляция и безопасность
Отдельный контур; Zero Trust; RBAC; секреты в Vault; синтетические данные не смешиваются с реальными
(Simulation Security / Exercise Data Governance Standards).

## 7. Связи

Framework · ADR-026 · GIS/Resource (Stage 6) · Technology Mapping (`CAP-DTWIN-001`).
