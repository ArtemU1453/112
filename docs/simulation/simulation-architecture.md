# Simulation Architecture — ЕАСУР

- **Status:** APPROVED · Stage 12 · Подчиняется Architecture.md, ADR-026, Simulation Governance /
  Validation / Scenario Modeling Standards. Изолированный контур.

## 1. Назначение
Механизм моделирования: движок сценариев ЧС и платформа симуляции (12.2, 12.3). Запуск сценариев,
изменение параметров, воспроизведение событий, ускорение времени, многократное повторение, сравнение
результатов.

## 2. Компоненты
| Компонент | Функция | Стандарт |
|-----------|---------|----------|
| Scenario Engine | Определение/загрузка сценариев ЧС (типы и комбинированные) | Scenario Modeling / Versioning |
| Simulation Runtime | Исполнение модели во времени (дискретно-событийная) | Simulation Validation |
| Time Controller | Реальное/ускоренное/пошаговое время | Simulation Governance |
| Parameterization | Изменение параметров прогона, seed | Scenario Versioning |
| Run Orchestrator | Одиночные и серийные прогоны (batch), повторяемость | Simulation Governance |
| Result Collector | Сбор метрик/событий прогона | Simulation Analytics |
| Comparison | Сравнение результатов прогонов/сценариев | Simulation Analytics |

## 3. Модель исполнения
Дискретно-событийная симуляция над состоянием двойника: события сценария → реакция ресурсов
(диспетчеризация/маршрутизация модельно) → изменение состояния → метрики. Детерминизм при фиксированном
seed (воспроизводимость, Simulation Validation Standard).

## 4. Типы сценариев (12.2)
Пожары · ДТП · аварии · химические происшествия · природные ЧС · техногенные аварии · массовые
происшествия · комбинированные события. Классификация согласована с доменом инцидентов (Stage 5).

## 5. Воспроизводимость и сравнение
Каждый прогон фиксирует: версию двойника, версию сценария, параметры, seed, наборы синтетических данных.
Повторный прогон с теми же входами даёт идентичный результат; серии прогонов → статистика (сравнение).

## 6. Изоляция
Симуляция работает только в контуре двойника (ADR-026); влияния на прод нет; данные — синтетические.
Перенос выводов в прод — только через Experiment Approval (RFC→ARB→ADR).
