# Runbook — Реагирование на эксплуатационный инцидент

- **Подчиняется:** Platform Operations Framework (Incident Management).

## Стадии
1. **Регистрация** — зафиксировать симптом, время, окружение.
2. **Классификация** — категория (доступность/производительность/безопасность/данные).
3. **Приоритизация** — влияние на реагирование 112 (critical/high/…).
4. **Назначение** — ответственная роль (POF Operational Model).
5. **Устранение** — по профильному runbook.
6. **Проверка** — health/метрики/трассы восстановлены.
7. **Закрытие** — уведомление, фиксация.
8. **RCA** — анализ первопричины; меры → задачи Backlog (техдолг/риски).

## Первичная диагностика
```bash
kubectl get pods -A --field-selector=status.phase!=Running
# метрики: Grafana ; логи: Loki (по correlationId) ; трассы: Tempo (по traceId)
```

## Эскалация
Incident Manager координирует; профильные администраторы (Platform/DB/Network/Security)
подключаются по зоне ответственности. Критичные инциденты безопасности — по SECURITY.md.
