# Message Platform — ЕАСУР

- **Status:** APPROVED · Stage 3 · ADR-008 · Подчиняется Architecture.md.

## Архитектура обмена
Apache Kafka (`CAP-MSG-001`) как событийная шина. Топики и потоки — ADR-008, System Structure.

## Очереди / события
Лог-ориентированные топики с партиционированием; ключ партиции задаёт порядок. Consumer-группы
обеспечивают fan-out (incident/dispatch/notification/realtime/audit).

## Гарантии доставки
Продюсеры — идемпотентны (`acks=all`); consumers — идемпотентны и толерантны к повторам/неизвестным
полям (at-least-once). Порядок — в пределах ключа партиции.

## Обработка ошибок и повторная доставка
Повторная обработка при сбое консьюмера; проблемные сообщения — по стратегии повторов/карантина
(dead-letter при необходимости, вводится через ADR при потребности). Мониторинг consumer lag
(Observability).

## Эволюция схем
Аддитивно; несовместимые изменения — новый топик с версией (AGS/DGS, ADR-008).
