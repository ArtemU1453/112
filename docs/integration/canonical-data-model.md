# Canonical Data Model (National) — ЕАСУР

- **Status:** APPROVED · Stage 9 · Реализует Canonical Data Model Standard (Stage 4) для
  межведомственного обмена. Подчиняется Canonical Identifier Standard.

## Назначение (9.5)
Единый канонический формат данных для обмена с внешними системами: справочники, идентификаторы,
словарь данных, правила преобразования, совместимость версий. Внутренние модели контекстов не
экспонируются напрямую (слабая связанность).

## Состав
- **Справочники (НСИ):** типы происшествий, службы, регионы, нормативы, статусы — эталонные,
  версионируемые (Reference Data Management Standard).
- **Идентификаторы:** канонические (Canonical Identifier Standard) — стабильные, уникальные,
  прослеживаемые; сопоставление внешних идентификаторов с каноническими.
- **Словарь данных:** канонические сущности (инцидент, вызов, подразделение, адрес, координата,
  сообщение) с типами/семантикой.
- **Правила преобразования:** внутренняя↔каноническая↔внешняя модель на границах (адаптеры/маперы).

## Совместимость версий
Канонический формат версионируется (SemVer); эволюция — аддитивно; несовместимо — новая версия с
планом совместимости. Схемы событий — Event Schema Governance Standard.

## Правила
Обмен только в описанных (канонических) форматах (запрет неописанных форматов); преобразования
документированы и версионируются.
