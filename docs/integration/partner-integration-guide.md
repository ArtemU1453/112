# Partner Integration Guide — ЕАСУР

- **Status:** APPROVED · Stage 9 · Подчиняется Partner Onboarding / Integration Certification
  Standards. Руководство по подключению внешних систем.

## Процесс подключения (9.9/9.10)
1. **Заявка и регистрация** в реестре внешних систем (владелец/назначение/интерфейсы/критичность).
2. **Согласование контрактов** (Contract First): выбор/версия API из каталога; канонический формат.
3. **Безопасность:** выпуск сертификатов (PKI), настройка mTLS, OAuth2 client credentials, scope
   минимальных привилегий (ADR-023).
4. **Песочница (Partner Sandbox, 9.9):** тестовые данные, эмуляция сервисов, **contract testing**,
   проверка совместимости.
5. **Сертификация интеграции** (Integration Certification Standard): соответствие контракту/
   безопасности/SLA.
6. **Продуктив:** подключение через платформу (прямой обход запрещён); мониторинг и аудит.

## Обязательства партнёра
Соблюдение версий контрактов и обратной совместимости; безопасный канал; идемпотентность;
уведомление об изменениях через RFC-процесс платформы.

## Отзыв/вывод
Отзыв доступа и вывод интеграции из эксплуатации — через реестр и governance (9.10) с аудитом.

## Definition of Done
Описан сквозной процесс подключения: регистрация → контракты → безопасность → песочница →
сертификация → продуктив → отзыв.
