# External Systems Registry Specification — ЕАСУР

- **Status:** APPROVED · Stage 9 · Подчиняется National Integration Architecture, Integration
  Governance (Stage 4).

## Назначение (9.2)
Реестр внешних систем — единый источник истины о подключённых информационных системах. Подключение
внешней системы без регистрации запрещено (запрет этапа).

## Атрибуты записи
Для каждой внешней системы:
- **уникальный идентификатор** (канонический);
- **владелец** (организация/ответственный);
- **назначение**;
- **интерфейсы** (используемые API/каналы);
- **версии контрактов**;
- **уровень критичности**;
- **требования безопасности** (mTLS/подпись/scope);
- **SLA** (доступность/латентность/поддержка);
- **журнал изменений** (append-only).

## Правила
- Доступ внешней системы предоставляется по записи реестра (scope/сертификаты); отзыв доступа —
  через реестр (9.10).
- Изменения записи версионируются и аудируются; критичность влияет на мониторинг/алерты.
- Реестр используется платформой (ADR-022) для маршрутизации/авторизации/мониторинга.

## Контракт
`contracts/admin/external-systems-registry-api.yaml` (регистрация/обновление/отзыв).

## Definition of Done
Определены атрибуты записи, правила доступа/отзыва, версионирование и связь с платформой/безопасностью.
