# API Governance Guide — ЕАСУР

- **Status:** APPROVED · Stage 9 · Дополняет API Governance Standard (AGS) и API Lifecycle
  Management Standard (Stage 4) для межведомственных API.

## Принципы
API First, Contract First. Контракт (OpenAPI 3.1 / gRPC proto) — до реализации; источник истины
для клиентов. Внешние контракты — канонические (Canonical Data Model) и версионируемые.

## Жизненный цикл API (9.3)
```
Design → Publish → Version → Test → Operate → Deprecate → Retire
```
Каталог API (реестр опубликованных интерфейсов, версий, владельцев). Публикация — только после
утверждения контракта и contract testing (Contract Testing Standard).

## Версионирование (API Versioning Standard)
Версия в пути/схеме; аддитивные изменения — без смены версии; несовместимые — новая мажорная
версия с планом совместимости. Одновременно поддерживается ограниченное число мажорных версий;
депрекация объявляется заранее (SLA/сроки).

## Категории
Internal (Stage 4/5/6/7/8) · Admin · **External/Interagency** (Stage 9, `contracts/external`).
Внешние — только через интеграционную платформу и реестр; mTLS/подпись (ADR-023).

## Правила (запреты этапа)
- Изменение API без новой версии/плана совместимости — запрещено.
- Неописанные форматы сообщений — запрещены (Contract First).
- Подключение внешних систем без регистрации — запрещено.

## Definition of Done
Определены принципы, жизненный цикл, версионирование, каталог и правила для межведомственных API.
