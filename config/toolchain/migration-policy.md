# Toolchain Migration Policy — ЕАСУР

- **Status:** APPROVED · Baseline: v1.0
- Часть Toolchain Version Management Standard (TVMS). Управляет обновлением версий из
  `versions.yaml`.

## Правила обновления версий

1. Изменение любой версии в `versions.yaml` — только через управляемый процесс (не «по месту»).
2. Обновление, влияющее на архитектуру или процесс разработки, требует ADR (Toolchain
   Governance Standard, TGS). Патч-обновления инструментов внутри совместимого диапазона —
   без ADR, но с записью в журнал изменений.
3. Категория версии (LTS/Supported/Preview/Experimental/Deprecated/EndOfLife) учитывается при
   принятии решения; переход на Preview/Experimental в prod-путь запрещён без RFC.

## Анализ совместимости

Перед обновлением:
- сверить `compatibility-matrix.yaml` (совместимые/несовместимые версии, зависимости `requires`);
- определить затронутые компоненты (сервисы, образы, CI, окружения);
- оценить транзитивное влияние (например, Spring Boot → Java, Kubernetes → Helm).

## Тестирование после обновления

- Полная сборка в чистом окружении (Dev Container/CI).
- Прогон всех уровней тестов (Unit, Integration/Testcontainers, Contract), релевантных изменению.
- Проверка воспроизводимости и прохождения Quality Gates.
- Проверка соответствия окружений (Environment Governance Standard).

## Стратегия отката

- Откат = возврат прежних значений в `versions.yaml` и связанных конфигурациях из системы
  контроля версий.
- Критерий отката: неустранимая несовместимость, регресс тестов/производительности, уязвимость.
- После отката — запись в журнал изменений и (при необходимости) задача Backlog/Technical Debt.

## Критерии завершения миграции

- Все затронутые сборки зелёные в чистом окружении.
- `compatibility-matrix.yaml` и (при снятии версии) `deprecated-versions.yaml` обновлены.
- Документация синхронизирована; CI-проверка версий проходит.
- Запись внесена в журнал изменений каталога.
